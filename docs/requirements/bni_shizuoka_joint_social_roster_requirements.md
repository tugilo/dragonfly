**想定開催日:** 2026-07-09（Google Sheets タイトルおよび既存議事録より。開始・終了時刻は TODO）  
**入力元:** [Google Sheets: 2026_07_09_BNI合同懇親会_参加者一覧](https://docs.google.com/spreadsheets/d/1rdGe1exX9Wcab5zx9a0n-955C4RiUCKY0HGJggJ6v_k/edit?usp=sharing)

## 1. 目的

Google フォームで募集した参加者情報をもとに、当日の交流で使いやすい参加者名簿を作成する。

目的は次の 2 つ。

1. **印刷配布・受付確認用:** A4 サイズで見やすく印刷できる名簿を作る。
2. **当日スマホ閲覧用:** 紙を持っていない場面でも、スマホから参加者を確認できるようにする。

---

## 2. 入力データ

Google Sheets のフォーム回答を入力元とする。確認できた列は以下。

| フォーム列 | 用途 | 名簿での扱い |
|------------|------|--------------|
| タイムスタンプ | 回答日時 | 重複時の新旧判定に使用。名簿には原則表示しない |
| 1. 名前 | 参加者名 | 表示必須 |
| 2. メールアドレス | 連絡先 | 表示必須。ただし個人情報として公開範囲に注意 |
| 3. 所属チャプター | 所属チャプター | 表示・グルーピングに使用 |
| 3-1. その他チャプター名 | 「その他」選択時の実チャプター名 | `所属チャプター = その他` の場合にチャプター名として使用 |
| 4. カテゴリ | BNI カテゴリ | 表示必須 |
| 5. 会社名 | 会社名 | 今回の表示項目には含めない。必要なら補足項目として追加検討 |
| 6. 一言 | 参加者の一言・つながりたい相手 | 表示必須 |

### 2.1 実効列の解釈

Google フォームの分岐により、`所属チャプター = その他` の行と、それ以外の行で表示上の列位置がずれる可能性がある。

名簿生成時は、次の「実効値」に正規化してから扱う。

| 実効項目 | 変換ルール |
|----------|------------|
| チャプター名 | `所属チャプター` が `その他` の場合は `その他チャプター名`、それ以外は `所属チャプター` |
| カテゴリ名 | `その他` 行ではカテゴリ列を使用。その他以外の行で列ずれがある場合は、実データ確認後に正しい列を採用 |
| 名前 | 前後空白を除去して表示 |
| メールアドレス | 前後空白を除去。メールリンク化する場合は `mailto:` に使用 |
| 一言 | 前後空白を除去し、A4 では折り返し・行数制限を行う |

## 3. 出力要件

### 3.1 A4 印刷用名簿

#### 目的

受付・当日配布・交流中の確認に使える、A4 サイズの参加者名簿を作成する。

#### 表示項目

ユーザー指定に従い、以下を表示する。

| 項目 | 必須 | 備考 |
|------|------|------|
| チャプター名 | 必須 | グループ見出しにも表示。行にも表示するかはレイアウトで判断 |
| カテゴリ名 | 必須 | 長い場合は折り返し |
| 名前 | 必須 | 参加者名順のキー |
| メールアドレス | 必須 | 印刷時は文字列表示 |
| 一言 | 必須 | 長文は折り返し。必要なら最大 2〜3 行で省略 |

#### 並び順

1. チャプター単位でグルーピングする。
2. チャプターは **参加者数が多い順** に並べる。
3. 参加者数が同数のチャプターは、チャプター名の昇順で並べる。
4. 各チャプター内の参加者は、**参加者名の昇順** で並べる。

#### レイアウト方針

- A4 横向きを第一候補とする。メールアドレスと一言が長いため、A4 縦より視認性が高い。
- チャプター見出しに `チャプター名（N名）` を表示する。
- ページ上部にイベント名、作成日時、入力元の更新日時または回答件数を表示する。
- 一言は交流のきっかけになるため残す。ただし紙幅を超える場合は、A4 では省略し、スマホ版で全文確認できるようにする。
- メールアドレスを含むため、配布範囲は参加者・運営関係者に限定する。

---

### 3.2 スマホ閲覧用名簿

#### 目的

当日、参加者がスマホから見られる名簿を用意する。A4 PDF をそのままスマホで見るのではなく、スマホ向けに読みやすい形式を別途用意する。

#### 表示方針

- チャプターごとに折りたたみ可能なカード形式を第一候補とする。
- 各参加者は「名前」「カテゴリ」「メールアドレス」「一言」をカード表示する。
- メールアドレスはタップでメール作成できるよう `mailto:` リンク化する。
- チャプター見出しに参加者数を表示する。
- 検索機能は任意。最低限、ブラウザのページ内検索で名前・カテゴリ・一言を探せること。

#### 共有方法

候補は以下。

| 案 | 内容 | メリット | 注意点 |
|----|------|----------|--------|
| A | レスポンシブ HTML を生成し、限定共有 URL で閲覧 | スマホで見やすい。印刷用と同じデータから作れる | 公開範囲・検索エンジン除外に注意 |
| B | Google Sheets の閲覧用シートを整形して共有 | 追加実装が少ない | スマホでは横スクロールが多くなりやすい |
| C | PDF をスマホでも共有 | 生成が簡単 | スマホ閲覧性が低い。今回は第一候補にしない |

現時点の推奨は **案 A: レスポンシブ HTML**。A4 PDF とスマホ HTML を同一の正規化データから生成する。

## 4. データ整形要件

| ID | 要件 |
|----|------|
| D1 | 名前が空の行は名簿対象外とする |
| D2 | チャプター名・カテゴリ名・名前・メールアドレス・一言は前後空白を除去する |
| D3 | `所属チャプター = その他` の場合、実際のチャプター名は `その他チャプター名` を採用する |
| D4 | 同一人物と思われる重複回答は、メールアドレス + 名前で判定し、原則として最新タイムスタンプを採用する |
| D5 | チャプター表記ゆれは勝手に統合しない。統合する場合は変換表を明示する |
| D6 | メールアドレスが空の行は残すが、要確認として扱う |
| D7 | 一言が空の行は空欄表示とし、参加者自体は除外しない |

---

## 5. 個人情報・公開範囲

メールアドレスを含むため、名簿は個人情報を含む資料として扱う。

- 印刷版は当日参加者・運営関係者への配布に限定する。
- スマホ版は、URL を知っている人だけが見られる形を基本とする。
- Web 公開する場合は検索エンジンに載らない設定を行う。
- イベント終了後の公開継続期間を決める。未決の場合は TODO とする。
- Google Sheets 原本の共有権限は、編集者を運営関係者に限定する。

## 6. 非スコープ

今回は「要件整理」と「名簿生成方針」の定義までを対象とする。

- Religo DB への参加者登録
- BNI メンバーマスタとの自動照合
- QR コード付き名札の生成
- 出欠管理・受付チェックイン機能
- 参加者本人による情報修正フォーム
- 懇親会後のフォローアップメール送信

次フェーズで作成する場合の成果物候補。

| 成果物 | 内容 |
|--------|------|
| A4 印刷用 PDF | チャプター別・参加者名順の名簿 |

## 8. 受け入れ条件

- [ ] Google Sheets の回答から、空行を除いた参加者だけが名簿に出る。
- [ ] チャプターは参加者数が多い順に並ぶ。
- [ ] 各チャプター内で参加者名順に並ぶ。
- [ ] A4 印刷版に「チャプター名・カテゴリ名・名前・メールアドレス・一言」が表示される。
- [ ] スマホ版でも同じ情報を読める。
- [ ] `その他` 選択者は、実際のその他チャプター名でグルーピングされる。
- [ ] メールアドレスを含む資料として、公開範囲が明示されている。
- [ ] 生成日時またはデータ更新日時が成果物に表示される。
