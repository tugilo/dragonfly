## 3. エデュケーション要点

### テーマ

**「紹介の解像度を上げる」**（担当：竹内俊太 — アスリート専門生命保険）

### 学び — 属性の三段階絞り込み

| 段階 | 例 | 評価 |
|------|-----|------|
| 第一段階 | 「経営者を紹介してください」 | ❌ 漠然としすぎて効果薄 |
| 第二段階 | 「都内で従業員10–30名規模のIT企業の経営者を紹介してください」 | △ 輪郭が見えてくる |
| 第三段階 | 「都内・従業員10–30名・IT企業の経営者で、最近オフィス移転または採用活動を始めた社長を紹介してください」 | ⭕ ピンポイントでヒット |

**なぜ重要か:** 75人体制（2,775ライン）の強力なインフラに、**具体的で高精度なデータ**を流し込むことで、最強のビジネスインフラになる。

### 顧客の悩み言語

顧客はサービス名ではなく、**日常会話の症状**で語る。

| ❌ サービス名で伝える | ⭕ 日常会話で出る言葉 |
|----------------------|----------------------|
| 「〇〇というサービスを探している」 | 「最近肩が痛くて眠れない」 |
| 「AI業務改善システムをしています」 | 「今年の税金が高すぎて頭が痛い」 |
| 「システム開発をしています」 | 「人を増やしたくないけど仕事が回らない」 |

**プレゼンの言い換え:** 商品説明ではなく —  
「今週あなたの周りで〇〇とぼやいている人がいたら、私につないでください」

## 4. メインプレゼン要点

### 木村健吾（Tシャツプリント）

#### 会社概要

- **株式会社ハートプランニング**（創業36年・岡山県倉敷市）
- 父の失明をきっかけに入社。47都道府県取引、最高一案件40,000枚

#### 強み

1. **何度でもデザインサポート** — 配色変更など修正対応
2. **追加注文も初回同額** — 少量追加でも単価維持
3. **1枚から作成可能** — シルク・転写・インクジェット・刺繍・レーザー加工を自社工場で一貫対応

#### 求めている紹介先

- スポーツチーム・クラブの監督・顧問
- 展示会・イベントを開く企業
- **特に:** 企業イベント企画担当（広告代理店業など）
- **夢:** BNI内最大規模「24時間テレビのチャリティーTシャツ」級の紹介

#### 印象に残ったポイント

- 1枚1,000–2,000円程度。衣服ではなく「人をつなぎ、気持ちを高め、メッセージを伝えるツール」
- Tシャツ以外にパーカー・エプロン・トート・作業着・ボトル・マグ・作業ヘルメット等も対応

#### 印象に残ったポイント

- 国内生産率：1990年50.1% → 2025年1.4%。必要なのは「作る力」＋「伝える力」＋「選ばれる力」

- **将来の夢:** 両国国技館で同展を主催。BNIで仲間探しを加速

## 5. シェアストーリー・学び

**登壇者:** 小中氏（使って学ぶAIツールの情報発信）

### 売りたいもの（当初）

- AI研修・コンサルティング
- **課題:** 教える人が多い / 初対面で選ばれる動機がない / 高額で即決できない

### 入り口戦略

- フロントサービスを **月1,000円の情報発信** に変更
- 導入しやすい価格帯で日々の情報発信 → 「AIに詳しい人」「教え方がうまい人」の認知獲得

### 信頼構築方法

- リファーラルマーケティングの設計要素を分解して実践：
  - 誰がお客か / 何に困っているか / どう解決するか
  - 誰とつながっていて、どう紹介してもらうか（入り口設計）
  - つないでもらった後のカスタマージャーニー設計
- 「誰でもいいです」という曖昧なリクエストでは仕組みにならない

### BNIとの相性

- トレーニングコンテンツが豊富
- DragonFlyは50人の経営者が在籍、年間140件以上の紹介を受けられる
- インプット（学び）とアウトプット（実践）の両方が同じ場で回る → **PDCAを早く回せる**

## 7. tugilo視点での気づき

### 気づき

今回の教育と小中氏のシェアストーリーは、**同じ構造**を指している。

- 竹内氏：紹介リクエストを三段階で具体化する
- 小中氏：高額サービスの前に、月1,000円の入り口で信頼を積む
- 共通点：**「何を売っているか」ではなく「どんな状況の人か」を言語化する**

75人体制の2,775ラインは「配線」。精度の低いリクエストを流すと配線は活きない。**悩み言語 × 具体属性**が、tugiloの紹介精度を決める。

→ 詳細は [Living Document §10](../../strategy/networking/BNI_Tsugihiro_Atsushi_Intro_Living_Document.md#10-bni活用とビジネス設計2026-05-26-定例会からの学び) に整理済み。

### 自社（tugilo）へ応用できそうなこと

1. **ウィークリープレゼン（25秒）** — カテゴリー名ではなく「困りごと」を先に言う
2. **1to1の入り口設計** — 相手に「私の紹介先、もっと具体化するなら何が足りない？」と聞く
3. **フロント商品の再確認** — 小中氏モデルのように、高額伴走の前段に「触れやすい接点」を置く
4. **石原氏フォロー** — 保険×介護福祉×地域イベント。経営者ネットワークと業務改善の接点を1to1で探索

### AI業務改善への置き換え例

| 今まで | 今後 |
|--------|------|
| 「AI業務改善システムをしています」 | 「最近、人を増やしたくないのに仕事が回らないと言っている社長いませんか？」 |
| 「Excelの二重入力を解決します」 | 「同じ内容を何度も入力しているとぼやいている担当者、周りにいませんか？」 |
| 「システム開発をしています」 | 「売上は伸びているのに、現場の仕組みが追いついていない会社の社長を紹介してください」 |
| 「経営者を紹介してください」 | 「従業員10–30名・IT以外の製造・小売で、採用または事務負担を最近増やした社長を紹介してください」 |

## 9. 一言まとめ

**「誰を紹介してほしいか」ではなく、「どんな悩みを口にする人か」を共有することで、紹介精度は大きく変わる。**  
チャプターの拡大は配線を増やすだけ。tugiloが次にやるべきは、75人体制に流し込む **高精度の悩み言語** を自分の言葉で定義し、毎週25秒で繰り返すこと。
