## 1. 背景

木村様の会社では現在、以下のオリジナル商品の **製造・販売** を行っている。

- デニム雑貨
- Tシャツプリント

### 会社体制

| 区分   | 人数   |
|--------|--------|
| 従業員 | 約30名 |
| 工場   | 約15名 |
| 店舗   | 約15名 |

### 現状の業務

- 受注
- 商品仕様
- 製造指示
- 出荷
- 顧客管理  

→ **紙ベース中心** で運用されている。

紙運用でも現場は回っているが、今後は次の点が **担当者の経験や記憶に依存しやすい状態** になっている可能性がある。

- 情報共有
- 仕様変更履歴
- 製造進捗
- 在庫状況

---

## 2. 本提案の目的

**紙の業務を単にデジタル化することではない。**

- 業務の流れを整理し、**会社全体の情報を誰でも把握できる状態** を作ることを目指す。

その結果として、以下につながる業務基盤を構築する。

- 製造ミスの削減
- 納期管理の向上
- 情報共有の効率化
- 将来の事業拡大への対応

---

## 3. tugiloの基本的な考え方

業務改善は次のように考える。

```
紙をデジタルにする → ではなく
業務を構造化する  → ことで実現する
```

- システムは目的ではなく、**業務をより良くするための手段** と考える。

---

## 4. 現状の業務構造（仮説）

今回のヒアリング内容から、業務の流れはおおむね以下の構造と考えられる。

```
受注 → 製造指示 → 製造 → 出荷 → 販売
```

※ 出荷には **2つの流れ** が存在する可能性がある（次節参照）。

---

## 5. 出荷の2つのパターン

### ① クライアント納品

企業や顧客から注文を受けて製造する商品。

```
受注 → 製造 → 出荷 → 納品
```

| 項目     | 内容           |
|----------|----------------|
| 特徴     | オーダーメイド |
| 出荷の意味 | 出荷＝売上     |

---

### ② 店舗補充

自社店舗へ商品を出荷するケース。

```
製造 → 店舗出荷 → 店舗在庫 → 店舗販売
```

| 項目     | 内容             |
|----------|------------------|
| 特徴     | 出荷＝在庫移動   |
| 売上計上 | 店舗POS          |

---

## 6. ペルソナ整理（業務ベース）

### ① 受注管理（事務）

| 項目     | 内容 |
|----------|------|
| 役割     | 受注内容整理、顧客対応、納期管理 |
| 課題（想定） | 紙注文、電話注文、仕様変更の共有 |
| 導入後   | 受注情報一元管理、製造指示生成、履歴確認 |

---

### ② 製造（工場）

| 項目     | 内容 |
|----------|------|
| 役割     | 製造作業、作業進捗管理 |
| 課題（想定） | 紙指示書、作業進捗共有 |
| 導入後   | タブレットで指示確認、進捗更新、作業履歴管理 |

---

### ③ 出荷管理

| 項目   | 内容 |
|--------|------|
| 役割   | 出荷準備、送り状、出荷記録 |
| 備考   | 事務または工場の担当の可能性 |
| 導入後 | 出荷登録、出荷履歴、受注連携 |

---

### ④ 店舗（将来拡張）

- 現在 **Airレジ** を導入。
- 利用状況: POS、売上管理。
- 在庫管理・顧客管理は未活用の可能性あり。

---

## 7. 将来的な店舗連携（オプション）

Airレジとシステムを連携することで、次のような **店舗と工場をつなぐ仕組み** も将来的に構築可能。

```
店舗販売 → 在庫減少 → 在庫アラート → 工場製造
```

※ 本提案では **将来構想** として扱う。

---

## 8. システム導入の基本方針

最初から大規模なシステムを作るのではなく、**段階的導入** を行う。

| 理由 |
|------|
| 現場負担を減らす |
| 教育コストを抑える |
| システム定着を優先 |

---

## 9. フェーズ構想

### Phase1: 受注・顧客・商品管理

- **機能:** 受注管理、顧客管理、商品管理、オーダーメイド仕様管理
- **できるようになること:** 受注情報共有、製造指示生成、履歴確認

---

### Phase2: 製造工程管理

- **例:** 生地準備 → プリント → 縫製 → 検品 → 出荷
- **できるようになること:** 製造進捗確認、納期管理

---

### Phase3: 在庫管理

- **対象:** 原材料、半製品、完成品
- **できるようになること:** 在庫状況確認、製造計画

---

### Phase4: 店舗連携

- Airレジ連携

---

## 10. 現場負担を減らす設計

導入時に重視するポイント。

- タブレット操作
- タッチ入力
- シンプルな画面

**目的:** 高齢のスタッフでも操作しやすい環境にする。

---

## 11. 将来的な業務の姿

### 受注情報の共有

```
受注入力 → 工場共有 → 製造開始
```

### 製造進捗の見える化

```
受注 → 製造 → 出荷
```

→ 進捗が確認できる。

### 商品仕様履歴管理

オーダーメイド商品のサイズ・カラー・プリントなどの履歴管理。

### 在庫把握

- 原材料・商品の在庫状況が把握できる。

---

## 12. 提案のスタンス

今回の提案は **システム導入を前提としたものではない**。

- まず **業務整理** と **課題把握** を行い、
- **どこから改善するのが最も効果的か** を一緒に考えていくことを目的とする。
