### KATANA（刀）の概要

#### 開発背景と目的

打田氏は、孤児院育ちという自身の背景から「生きた証を残したい」という思いを持っている。複数事業の経営経験を通じて、情報の即時可視化、人材評価、経営者が数字と向き合わない文化に課題を感じたことが、KATANA 開発のきっかけになった。

最終的には、日本企業が永続的に繁栄し、人に時間とお金を提供できる仕組みを作ることを目指している。

#### 主要機能

| 領域 | 内容 |
|------|------|
| 財務・会計 | PL、貸借対照表、キャッシュフロー、今期着地予想、総勘定元帳をリアルタイム生成。口座情報と連動し現預金を管理 |
| 税務連携 | 税理士への確認段階まで自動化。技術的には自動申告まで可能だが、税理士業務を残すため意図的に制限 |
| 案件・プロジェクト管理 | 案件進捗、残工数、参加メンバー、見積もりベースの品目別時間配分を一元管理 |
| 商談管理 | 商談状況、契約者、打ち合わせ議事録、受注状況を追跡 |
| 勤怠・業務報告 | 出退勤時に案件別・品目別の作業内容と使用時間を報告。約5分で完了する設計 |
| 人事評価・利益分析 | 社員ごとの売上貢献度、粗利率、生産性、時間あたり利益を実数値で算出 |
| 権限管理 | 画面ごとの閲覧・編集・非表示を細かく設定。役職や業務内容に応じた表示が可能 |
| AI機能 | 現在は使い方案内・伴走支援レベル。将来的には意思決定や交渉まで行う知能へ育てる構想 |
| 社内コミュニケーション | カレンダー、社内チャットを搭載。外部に情報が漏れにくい設計 |

#### 開発期間と現状

- 開発期間は約3年半。
- 2026-05-07 リリース、2026-05-20頃から営業開始。
- 代理店契約は約30社、導入アカウントは約50件。
- 業種構成は、士業20%、営業会社30%、建設業15%、その他サロン・店舗等。
- 特に評価されているのは、商談・プロジェクト管理機能と、現場作業員の行動可視化機能。

### 次廣からの評価とフィードバック

#### 高評価ポイント

次廣は、経営サイドの統合システムとして機能が揃っている点を高く評価した。特に、決算処理まで自動化できること、案件管理・見積もり・請求管理を一つにまとめたこと、3年半でこの規模のシステムをリリースしたことを大きな成果と捉えた。

#### 改善提案 1: ユーザーインターフェース

現状の画面は「エンジニア目線」で、使う側、特に一般社員の視点が不足している可能性があると指摘した。打田氏が直接話すと伝わる熱意や思想が、システム画面上ではまだ十分に伝わっていない。

数字を並べるだけでは理解できない人も多いため、インタラクティブな表示、ビジュアル化、楽しさを感じる導線が必要。例として、Amazonの工場で使われている「仕分け作業をするとたまごっちが育つ」ようなゲーミフィケーションを挙げ、入力行為そのものにベネフィットを感じさせる工夫が重要だと提案した。

#### 改善提案 2: 機能の疎結合化

「何でもできる」ことは強みである一方、企業によっては使わない機能が出る。請求・見積もり・売上管理だけ必要な企業もあるため、機能ごとに選択できる疎結合設計やコース別提供にすると、より売りやすくなる可能性がある。

#### 改善提案 3: データ移行

既存システムからのデータ移行について確認し、CSV形式での取り込みが可能であることを把握した。API接続はセキュリティ面で課題があるため現時点ではCSV対応だが、過去データ投入やシステム切替の導線は作れる。

#### 改善提案 4: AI導入と評価制度

企業はAI導入を望んでいるが、使い方が分からない状態にある。KATANA は、AI導入補助金やIT導入補助金との相性が良い可能性がある。

AI時代の役割分担としては、AIが提案まで行い、最終判断は人間が行う形が現実的。人事評価については、時給ベースから成果物ベースへ移行する必要があり、KATANA が持つ項目別の生産量・時間あたり利益の可視化は、その転換に使える。

### 打田氏のビジョンと課題認識

#### 日本企業の構造改革

打田氏は、日本の中小企業が衰退している根本原因として、固定給・年功序列制度や、経営者が数字と向き合わない文化を挙げた。現役世代の経営者が改革の主体になる必要があるという認識。

毎年多くの会社が新設される一方で、半数程度しか残らない現状を問題視している。スタートアップ段階からシステムを導入し、社員を巻き込んで潰れる企業を減らす動線を作ることが重要だと語った。

#### 人材育成と実数値評価

能力が伸び悩む社員の問題は、企業がロードマップを実数値で示せていないことにもある。業務報告を細かく取ることで、事実ベースの数字を定規にしながら社員と話し合い、育成につなげる構想。

中間管理職が報告を転記するのではなく、現場作業員が直感的に事実ベースで入力するだけで、評価と育成に使える情報が集まる状態を目指している。

#### 長期ビジョン

打田氏は現在の KATANA を「0.5段」と位置づけている。全機能の統合とAIの育成が進めば、最終的にはAIだけのUIで経営が回る、無人化経営に近い状態を目指す。

さらに長期的には、2035年から宇宙開発計画を開始し、地球外から物資を調達する構想も語った。KATANA はその大きな構想に向けた初期段階の基盤という位置づけ。

#### 現在の課題

打田氏自身の課題認識は、影響力のある人を巻き込むマーケティング力と、開発資金を継続的に調達する財務力。自身が代表であり続けることにはこだわらず、思想を引き継ぎ、影響力を持つ人が広めてくれる状態を望んでいる。
