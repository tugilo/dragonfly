### 朝礼スライド作成フロー

1. **締切:** 朝 **9 時まで** にアップロードされたファイルを対象とする
2. **ファイル取得:** [NextCloud ST定例会 共有フォルダ](https://ne-dragonfly.site/cloud/index.php/s/E3a37t3pPsAJB8q?path=%2F) の該当日付フォルダから各 ST のスライドをダウンロード

## 技術的な知見

### NextCloud 運用

- **共有 URL（ST定例会）:** [https://ne-dragonfly.site/cloud/index.php/s/E3a37t3pPsAJB8q?path=%2F](https://ne-dragonfly.site/cloud/index.php/s/E3a37t3pPsAJB8q?path=%2F)
- **自動フォルダ生成:** 毎週 **火曜 13 時** に次回開催日のフォルダが自動作成される
- **アーカイブ管理:** 第 5 期〜第 9 期はアーカイブ化、**第 10 期が現行**
- **例外処理:** モメンタム開催時などは **手動でフォルダ名を変更**

### スライド統合の課題と対策

| 課題 | 内容 |
|------|------|
| **混在問題** | 過去に複数回、異なるバージョンのファイルが混在する問題が発生 |
| **原因** | パワーポイントを複数立ち上げて **保存先を間違える** ヒューマンエラー |
| **対策（検討）** | **まっさらな状態から作成** する手順への変更 |

### クラウドコードによる自動化（倉持実装）

- **スライド統合:** クラウドコードの **Python** で自動化（処理に時間がかかる）
- **ノートセリフ抽出:** スライドの **ノート欄の台詞** から切り替えタイミングを判断
- **台本とスライドの整合性チェック:** 台本の数字とスライドの数字の **不一致を自動検出**（次廣も外部チェックツールを作成）
- **リリース方針:** **チャプターの誰でも使える** こと（改造は不要でも可）。過度な自動化による **属人化** は避ける

### Canva・プラットフォーム

| 項目 | 内容 |
|------|------|
| **Canva の課題** | AWS 米国リージョンで認証処理のため障害時に全世界で使用不可のリスク。PPT 直接エクスポートはレイアウト崩れ。**画像化して使用** |
| **表示問題** | Canva スライドに変な帯が表示される問題あり |
| **現状の選択** | ローカル編集可能な **パワーポイント** |
| **クラウド化候補** | **Google スライド** |
| **Canva 使用時** | 必ず **前日にダウンロード** しておく運用 |

### ワントゥワン・ネットワーキングデータベース

| 実装 | 内容 |
|------|------|
| **次廣** | Zoom から自動取得した議事録を **ローカル DB** に蓄積。メンバー情報・「誰と何を話したか」・つなげるべき人のリストアップを自動化。**ChatGPT 連携** でサーバー上でも同様の機能を実現可能 |
| **倉持** | **Obsidian** とクラウドコードでローカル DB を構築し、ワントゥワン内容を自動記録 |
| **共有構想** | 個人の 1to1 履歴を DragonFly メンバー全員で共有（**ブラックボックス化して推薦のみ出力** する形式を検討）。ローカル実装は完了、**クラウド化は未検討** |
| **次廣の方針** | **記憶力ではなく記録力** で勝負。ブレイクアウトルームの同席者も記録 |

## ビジネス戦略・協業構想（議論メモ）

### 海外進出支援

- ターゲット 100 社中 **2〜3 社程度** しか海外進出条件が揃わない
- 主な制約: 人材不足、生産能力不足、在庫リスク回避、流通経路の欠如
- 適合企業: 生産能力はあるが在庫を抱えたくない企業、または在庫を抱えているが売れていない企業

### ライブコマースの課題

- 日本企業はテストマーケティングでも **確実な結果** を求める傾向
- **30〜40 万円** という低価格でもテストとして受け入れにくい — **金額より未経験への心理的抵抗** が主因

### AI インフルエンサー戦略（倉持）

- 従来のライブコマース課題の解決策として開発
- 中国側協力: AI ドラマ無料提供、プロダクトプレイスメント広告可能
- レベニューシェア事前設定で無料宣伝も可能
- メリット: 24 時間稼働、スキャンダルリスクなし、分身可能、コスト削減
- **企業専用開発:** 月 **30,000 円** で人格からデザイン可能

### AI 導入支援の連携

- **次廣 × 小中:** 企業向けセキュア AI 構築
- **シャドー AI 問題:** 従業員の個人クラウド AI 利用による機密流出リスク → **ローカル AI サーバー** で安全な利用環境
- **タイミング:** 数ヶ月後に社会問題化する前の **先行対応**

### フロント商材・リストマーケティング

| テーマ | 内容 |
|------|------|
| **望月・就業規則無料診断** | AI で自動化し精度の高い診断と **助成金提案** |
| **次廣・提案書** | 現状分析と改善提案書を **無料提供** し成約率向上 |
| **倉持・配信** | 7,000 件名刺リストへ **パーソナライズ案内** |
| **リスト規模** | 約 7,000 件（**約 5 分の 4 が死んでいる可能性** → 開封追跡で復活計画） |
| **クラウドコード分析** | 名刺から企業リサーチ、送信可否を自動判断。**250 件を 2〜3 時間** で個別メッセージ作成 |
| **高度リサーチ** | プレスリリース参照、HTTPS 未対応サイト抽出＋動画説明送信等の実績 |
| **新アプローチ** | リストマーケを **リファーラルマーケ** の手法で実施 — 受信者が読んだ瞬間に可視性・信頼性が上がるレベルのパーソナライゼーション |
| **メルマガ** | 継続購読者リストが最強資産。配信・セグメント自動化、**PC 1 台常時起動** で配信スタンド不要 |

### パワーチーム・アライアンス

| 項目 | 内容 |
|------|------|
| **パワーチーム定義** | 同チャプター内でリファーラルを回し合うコミットメントチーム。公式 **月 2 回以上**、倉持推奨は **週 1 回**、30 分〜1 時間 |
| **倉持アライアンス** | 他チャプター・BNI 外も含む。**清水・福士** の 3 名で開始。お互いの送客の仕組み理解、フロント商材メンバー支援。倉持が **中国の窓口** |
