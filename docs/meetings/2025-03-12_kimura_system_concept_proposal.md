## 1. 現状の業務と課題

### 業務の全体像（仮説）

現在の業務の流れは、お話を伺う限りでは次のような構造になっているように見えます。

```
受注 → 製造指示 → 製造 → 出荷 → 販売
```

この流れ自体はとてもシンプルですが、  
実際の業務の中ではいくつかのパターンが混在しているようです。

---

### 出荷には2つの流れがある

打合せの中で、「出荷」の意味合いが場合によって異なることが見えてきました。  
この違いは、仕組みを整理する上でとても重要なポイントです。

#### ① クライアント納品（出荷 ＝ 売上確定）

```
受注 → 製造 → 出荷 → 納品
```

オーダーメイドの注文を製造・納品するケースです。  
この場合は**出荷の時点で売上が確定**します。

#### ② 店舗補充（出荷 ＝ 在庫の移動）

```
製造 → 店舗出荷 → 店舗在庫 → 店舗販売（Airレジ）
```

自社店舗への補充のケースです。  
この場合は**出荷は在庫の移動**であり、  
売上は**Airレジで計上**されます。

---

### 現状で気になるポイント（仮説）

現場は今日もちゃんと回っています。  
ただ、お話を伺う限りでは以下の点が

**担当者の経験や記憶に頼りやすい状態**

になっている可能性があります。

| 領域 | 気になるポイント |
|------|------------------|
| 受注・仕様 | 変更内容の共有が口頭・メモ中心になりやすい |
| 製造進捗 | 工場の外から現状が見えにくい |
| 仕様の履歴 | 変更の経緯が記録として残りにくい |
| 在庫状況 | リアルタイムでの把握が難しい |
| 情報共有 | タイムラグや伝達漏れが起きやすい |

これらは今すぐ大きな問題になるわけではありません。

ただ、

**会社が大きくなるほど**  
**担当者が変わるほど**

じわじわとリスクとして表れやすくなります。

## 2. 立場別の課題と改善イメージ

それぞれの立場で  
「もし仕組みが整うと、何が変わるか」を整理しました。

---

### 👔 経営（木村様）

| 項目 | 現状 | 改善後のイメージ |
|------|------|------------------|
| 受注状況の把握 | 担当者に都度確認が必要 | どこからでもリアルタイムで確認できる |
| 製造の進捗 | 工場に聞かないとわからない | 進捗・納期が一目で見える |
| 在庫の状況 | 現場感覚に依存している | データとして把握できる |
| 属人化のリスク | 担当者不在で業務が止まりうる | 情報が仕組みに乗り、誰でも動ける |

---

### 📋 事務（受注管理）

| 項目 | 現状 | 改善後のイメージ |
|------|------|------------------|
| 受注の受付 | 紙・電話・メールが混在 | 受注情報をひとつにまとめて管理 |
| 仕様変更の共有 | 口頭・メモ中心で漏れのリスク | 変更履歴が記録され工場へ即共有 |
| 製造指示 | 手書きの指示書を作成・届ける | 入力するだけで工場へ共有される |
| 納期管理 | 頭の中・手帳での管理 | 一覧で確認・見落としを防げる |

---

### 🏭 製造（工場）

| 項目 | 現状 | 改善後のイメージ |
|------|------|------------------|
| 作業指示の確認 | 紙の指示書を受け取る | タブレットでシンプルに確認できる |
| 仕様の確認 | 指示書を見返す・担当者に聞く | 最新仕様・変更履歴をその場で見られる |
| 進捗の共有 | 口頭・電話での報告 | 工程ごとに更新するだけで伝わる |
| 作業の記録 | 残りにくい | 自然に蓄積されていく |

> 操作はタブレット・タッチ入力を前提に、  
> シンプルな画面設計を優先します。  
> ベテランのスタッフでも無理なく使えることを大切にします。

---

### 🏪 店舗（将来的な構想）

| 項目 | 現状 | 将来のイメージ |
|------|------|----------------|
| 在庫の確認 | Airレジで販売管理のみ | 工場・店舗の在庫をまとめて把握 |
| 補充の依頼 | 都度連絡して確認 | 在庫が減ったら工場側に通知が届く |

> 店舗との連携は**将来の構想**です。  
> 今すぐ対応する必要はありません。

## 3. 段階的な仕組みづくりの構想

最初から大きく作る必要はありません。

**小さく始めて、効果を確認しながら積み上げていく**

という進め方が現実的だと考えています。

各フェーズは独立しており、

- Phase1から始める
- 現場が慣れてきたらPhase2へ

という進め方もできます。

---

|  | Phase1 | Phase2 | Phase3 | Phase4 |
|--|--------|--------|--------|--------|
| **テーマ** | 受注〜納品の一元管理 | 製造進捗の見える化 | 在庫の把握と管理 | 店舗連携（将来構想） |
| **効果** | 情報が全員に届く | 納期を先手で管理できる | 欠品・過剰在庫を防ぎやすくなる | 製造〜販売がつながる |

> どのフェーズから始めるのが最も現実的かは  
> 今後のヒアリングを通じて一緒に整理できればと思っています。

---

### Phase1：受注から納品まで、情報をひとつにまとめる

受注情報を入力するだけで工場に共有され、  
仕様変更の履歴も自動で残ります。

クライアントへの納品・出荷管理も  
ここで一元管理します。

> **「あの注文、今どこまで進んでる？」を確認しなくてもわかる状態へ。**

---

### Phase2：製造の進捗を、工場の外からも見えるようにする

```
生地 → プリント → 縫製 → 検品 → 出荷
```

それぞれの工程がどこまで進んでいるかを  
リアルタイムで確認できるようにします。

> **「納期、大丈夫かな」という不安を数字で確認できるようになります。**

---

### Phase3：在庫の状況を、感覚ではなくデータで把握する

- 原材料
- 半製品
- 完成品

の在庫状況をリアルタイムで確認できるようにします。

> **材料不足による製造停止や、店舗の在庫切れを防ぎやすくなります。**

---

### Phase4：店舗と工場をデータでつなぐ（将来構想）

Airレジと連携することで、次のような流れが作れる可能性があります。

```
店舗販売 → 在庫減少 → 工場へ通知 → 製造
```

---

## 4. 次のステップ

今回のお話では、まず全体像をざっくり共有いただいた段階だと感じています。

次のステップとしては、  
すぐに仕組みを決めるのではなく、

**木村様のお話をもう少し詳しくお聞きする時間**

をいただければと思っています。

特に次回は、私からの提案よりも

- 今どこに一番困りごとを感じているか
- 受注から出荷まで実際にはどんな流れで動いているか
- どこは紙のままが良くて、どこなら変えられそうか
- 将来的にどういう状態になると理想か

といった点を、木村様のお話を中心に深掘りできればと考えています。

---

| ステップ | 内容 |
|----------|------|
| ① 次回打合せ | 木村様のお話を中心に、現状の流れや困りごとを詳しく伺う |
| ② 整理・共有 | お聞きした内容を整理し、論点を見える化して共有する |
| ③ 今後の進め方を相談 | 必要であれば現場確認も含めて進め方を相談する |
