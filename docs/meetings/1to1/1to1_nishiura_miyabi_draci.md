#### ■ パートA — 3層アカウント管理体系

##### 背景・現状の課題

| 論点 | 内容 |
|------|------|
| **平岡氏のアカウント** | 現状、WordPress **管理画面全体** にアクセス可能 |
| **平岡氏の意向** | 管理作業は **一切したくない・触りたくない** |
| **実運用** | ログイン情報は **西浦さん** に渡され、西浦さんが実質的な管理を担当 |
| **次廣の懸念** | 以前から「平岡氏がこの権限状態で運用できるのか」と疑問を持っていた |

##### 合意した3層構成

| 層 | アカウント種別 | 権限・用途 |
|----|----------------|------------|
| **1** | **ドレスシー管理者アカウント**（平岡氏用） | 会員の **承認管理など限定的な機能のみ**。通常メンバーと同様の **ログイン専用** に変更 |
| **2** | **WordPress システム管理アカウント** | テーマ・設定など **技術的管理機能**。システムを動かす人専用。**1アカウントで十分**（複数人が同時ログインして作業可能） |
| **3** | **通常メンバーアカウント** | 一般ユーザー向けの標準アクセス。プロフィール自己編集・掲示板利用等 |

##### 決定事項

- **アカウント体系の分離:** 平岡氏のアカウントを通常メンバーと同様の **ログイン専用** にし、別途 **システム管理用アカウント** を作成する
- **管理画面へのアクセス制限:** 平岡氏は WordPress 管理画面を **一切触らない**。システム運用者向けログインを **別途用意**
- **管理者アカウント数:** システム管理用は **1つで十分**（複数人同時ログイン可）
- **作業の無償提供:** 提案書に基づく作業は **既存プロジェクトに含め、追加費用なし**
- **権限設計の考え方:** 管理画面は普段使わない想定。誤操作防止のため不要機能へのアクセスを制限

##### 会員登録フロー（現状）

5. **西浦さん** がユーザー作成〜パスワード発行まで担当。**平岡氏は登録プロセスに直接関与していない**

## ■ 累積インサイト

- **西浦さん = 実務のハブ:** 平岡代表名義でも、WordPress・会員登録の **実運用は西浦さん**。アカウント整理も **西浦さんとペア** で進めるのが自然。
- **平岡氏の権限最小化は合意済み:** 「触らせない」が正式な運用方針。提案書作業は **追加費用なし** でプロジェクト内対応。
- **掲示板は評価良好:** 早い進捗・プラグイン最小化が刺さっている。次の品質ゲートは **テストサーバー上の西浦さん FB**。
- **支部検索は将来拡張の布石:** 名称未確定だが、ユーザー属性＋紹介ページ UI の両方に載せる方針で来週実装。
- **LTST／旧 WordPress 資産:** 簡素化・引き継ぎ可能な設計への不満は継続テーマ。Draci 改善が **対照的な成功例** として位置づけられている。
- **Myria-mu 管理システムは新規案件化の可能性:** 西浦さん本人の外構SNS集客事業に直結する。顧客・施工店・インフルエンサー・報酬の4者関係を整理できれば、15〜18万円規模の初期提案として切り出せる。
- **専用URL管理が核:** クーポンコードではなくリンクで流入元を確定できると、インフルエンサー報酬計算と顧客進捗管理がつながる。
- **報酬率は要確認:** 通常3%・アンバサダー4%の理解で進めつつ、Zoom要約に逆表記があるため提案前に西浦さんへ確認する。
