## ■ サマリー（最新状況）

- 竹村裕司さんは、芳賀崇利さん紹介ゲスト。元大阪のGaiYenチャプターで芳賀さんと一緒だったメンバーで、現在は奈良県在住。13ヶ月の残存証明書があり、新たなチャプターを探している。
- ゲスト登録カテゴリーは **プロモーション動画制作**。動画制作からWeb制作までクリエイティブを幅広くサポートする。
- 竹村さんは DragonFly 入会に向け、株式会社ONODE全体ではなく **チアプリント** も紹介しやすい入口商材として打ち出す方針。
- チアプリントは、全国コンビニのプリントキオスクで写真・イラスト・資格系・学習系・シール等のコンテンツを販売できるサービス。掲載無料、高マージン還元、全国対応、即日販売開始が強み。
- 次廣側は、AI活用開発、汎用予約管理システム、BNI向けリファラル支援システム（Religo）の構想を共有。竹村さんから、サービスの売り出し方・アピール方法で協力を得られる可能性が確認された。
- 両者とも「何でもできます」型の事業で、BNIでは紹介しづらくなる課題を共有。入口商材を明確にする重要性で一致した。
- Dリーグ・ダンス業界への共通関心があり、次廣の娘のダンス講師経由の DYM メッセンジャーズ接点と、竹村さん側のロイヤルラッツ契約を起点に、チアプリント掲載の紹介余地がある。

#### Decisions & Agreements

#### 竹村さん側の事業理解

- ONODEは、制作物そのものに加え、費用対効果・広報運用まで見ていくプロモーション色の強い会社。
- LEDビジョンレンタルは、展示会・イベント・大型映像訴求と相性がよい。映像制作も合わせられるため、単なる機材貸しではなく「見せ方」まで支援できる。
- チアプリントは、コンビニ店頭のプリントキオスクを販売チャネルにできる。アイドル、ダンス、スポーツ、BL、学習系、資格系など、幅広いコンテンツ販売に対応する。
- 前チャプターでの経験から、BNIでは「ロゴも名刺も動画もWebもできます」より、**チアプリントという尖った紹介入口**を持つ方が伝わりやすいと判断している。

#### チアプリントの強み

| 観点 | 内容 |
|------|------|
| 掲載費 | コンテンツ登録・掲載は完全無料 |
| 還元率 | コンテンツ制作者に最大30%を還元。他社は10〜15%程度のケースが多い |
| 収益構造 | コンビニ＋印刷会社50%、ONODE側20%、制作者30% |
| 対応店舗 | ローソンを含む全国コンビニ約60,000店舗対応 |
| 競合比較 | イープリント、ピクシブ等は特定チェーン中心のケースがあるが、チアプリントは全コンビニ対応が強み |
| 開始スピード | 前事業者の契約内容を継承しており、審査なしで即日販売開始が可能。他社は1ヶ月程度かかる場合あり |
| 機能 | 写真、シール、資格系コンテンツ、ガチャ機能、シークレット機能などに対応 |
| 実績・需要 | 中高生は放課後（16〜19時）に集中アクセス。嵐のラストライブ関連で大きな売上実績あり |

#### 次廣側の共有

- 次廣はエンジニア歴26年のシステムエンジニアとして、B2Bの業務システムをフルスクラッチで開発している。
- Claude Code などのAIを活用し、コーディング作業を大幅に削減。今後は要件定義・ヒアリング・価値設計により注力する方針。
- 事業者目線の汎用予約管理システムを開発中。開店前の時間帯ブロック、最適な予約枠の自動提案など、現場運用に寄せたクラウド型システムを夏頃リリース予定。
- BNI向けには、1on1記録を蓄積し、AIがリファラル候補を提案する仕組みを開発中。Zoom文字起こし要約をコピペするだけで、誰と何を話したかの履歴を残す運用を想定。
- 他者の1to1内容を直接のぞくのではなく、システム側が「つながりの可能性」を提案する形でプライバシーに配慮する。
- A→B→C→A のように、メンバー全員がリファラルを出しやすくなる循環を作ることを目指す。

#### BNIにおけるポジショニング

- 共通課題として、両者とも「何でもできます」型で、BNIで紹介しづらい問題がある。
- 竹村さんは、チアプリントで入口を尖らせる方針。
- 次廣は、汎用的な入口商材として予約管理システムを作り、紹介しやすい形で展開する計画。
- 誰に何を紹介するかを明確にすることが、BNIでの成功の鍵だと両者で認識した。

#### Dリーグ・ダンス業界の接点

- 次廣の娘のダンス講師が、Dリーグチーム DYM メッセンジャーズのディレクター・ユウマ氏。

#### AI・システム開発トレンド

- AIにより80〜90%のシステムは誰でも作れる時代になったが、100%へ引き上げること、運用・保守を成立させることには専門知識が必要。

- 見積もりはコストベースからROIベースへ転換していく必要がある。
