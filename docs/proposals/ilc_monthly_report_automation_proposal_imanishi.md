## 1. 結論

いただいたデータと作業手順を確認した限り、**月次集計表作成のうち、TSV取得後の加工〜集計表作成部分はかなり自動化できる可能性が高い**です。

特に、現在 Excel への貼り付け、前回データ削除、日付補完、変換、転記を繰り返している部分は、プログラムで一括処理しやすい構造でした。

最初から大きなシステムを作るのではなく、まずは **AI駆動で小さく検証 → Web MVP → レポート生成版** の順で進めるのが現実的です。

## 2. 検証で分かったこと

手元データから、以下を確認しました。

| 項目 | 確認結果 |
|------|----------|
| 対象データ | チカラグループ / ファボローネ |
| TSV数 | チカラグループ 15ファイル、ファボローネ 36ファイル、合計 51ファイル |
| データ行数 | 合計 76,628 行 |
| 文字コード | `cp932` で読み取り可能 |
| TSV列 | `店名`, `日付`, `category`, `action`, `label`, `件数`, `時`, `分`, `秒` |
| 現行作業 | TSVをExcelへ貼り付け、削除、日付補完、変換、転記、最終表作成 |

列構造が揃っているため、日付・店舗・カテゴリ・アクション・件数をもとにした自動集計に向いています。

---

## 3. 自動化できること

まず自動化しやすいのは、以下の範囲です。

- TSVファイルの読み込み
- 文字コード変換
- 店舗別・日別・項目別の集計
- 日付欠損の補完
- Excel / CSV への出力
- 加盟店向けレポートの形式調整

一方、最初から無理に含めない方がよいものもあります。

- CMS管理画面からのTSVダウンロード自体
- 加盟店へのメール自動送信
- 宛先管理
- 送信履歴管理
- 詳細な権限管理

これらは後から拡張できます。初期段階では、**TSVをアップロードすれば集計表が出る** ところに絞るのが費用対効果の面で良いです。

| 時給換算 | 月次削減額 |
|----------|------------|
| 3,000円/時 | 約9.6〜10.8万円/月 |

## 6. 提案プラン

### プランA: AI検証プラン

概算: **10〜20万円（税別）**

代表店舗のTSVを使い、現行Excelと主要数値が合うかを確認します。

できること:

- TSV構造の確認
- 集計ロジックの試作
- 現行Excelとの数値照合
- 自動化できる範囲とできない範囲の整理

向いているケース:

- まず小さく検証したい
- いきなり開発費をかけず、実現性を確認したい
- 社内説明用の根拠を作りたい

### プランB: AI駆動Web MVP

概算: **80〜90万円（税別）**

ブラウザからTSVをアップロードし、店舗別・日別・項目別に集計したExcel / CSVを出力する最小実用版です。

含めるもの:

- TSVアップロード
- 店舗別・日別・項目別集計
- 日付欠損補完
- Excel / CSV 出力
- 最小限の画面

初期スコープから外すもの:

- 本格的なログイン機能
- 送信管理
- 宛先管理
- 高度な履歴管理

向いているケース:

- 毎月の手作業を早く減らしたい
- 費用を抑えて実用化したい
- 効果を見てから拡張したい

### プランC: AI駆動レポート生成版

概算: **160〜180万円（税別）**

加盟店に送る資料として見やすいExcel / PDFレポートを、Web上で自動生成する版です。

できること:

- 月次集計結果の自動出力
- 店舗ごとのレポート生成
- Excel / PDF 形式の出力
- 表中心・グラフ中心など複数の見せ方の検討
- 加盟店向けに見やすい資料化

向いているケース:

- 加盟店に送る資料の品質も上げたい
- 既存Excelの見た目にこだわらず、分かりやすく作り直したい
- 毎月の資料作成を標準化したい

## 7. 推奨する進め方

おすすめは、次の段階的な進め方です。

| Step | 内容 | 概算 |
|------|------|------|
| 1 | 代表店舗でAI駆動の集計ロジックを検証 | 10〜20万円 |
| 2 | TSVアップロードから集計表出力までのWeb MVP | 80〜90万円 |
| 3 | 加盟店向けExcel / PDFレポート生成 | 追加 80〜90万円 |

初回は、Step 1 または Step 1+2 までに留めるのが良いです。  
数字が合うこと、実際に作業時間が減ることを確認してから、レポート生成や送信管理へ拡張します。
