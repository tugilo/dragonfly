## 1. 背景（Why）

### 1.1 既存環境の前提

- 貴社で **WordPress を継承運用** されている。  
- **プラグインが多く、過去の修正で他箇所に影響**が出た経緯がある。  
- 現在の主な利用: **トップ、会員・会員一覧、掲示板、会員情報の自己編集**。  
- **決済・入会申込**周りは **現状未使用だが、将来のため** **すぐ削除前提にはしない**（触る範囲は**別途合意**）。

### 1.2 今回扱うテーマ（2本立てである理由）

- **案件A: 会員向け掲示板** … **機能として「ちゃんと使う」**（設計・画面・会員連携）。  
- **案件B: プラグイン整理** … **調査・分類・リスク管理**（**止める／留める**の**根拠**が成果）。  
- **見積・契約**を**分けないと**、遅延や追加要望が**片方の予算**を圧迫しやすい。

**一言**: 「新機能」と「家の中の配線図作り」は、**成果物の形とリスク**が違う。

---

## 2. 方針（What）

### 2.1 掲示板の位置づけ（誤解防止）

- 一般的な **チャット型掲示板** ではない。  
- **会員同士の「コラム・案件・知見の投稿」＋コメント**、**あとから検索・分類で辿る** 用途。  

### 2.2 技術方針（一次合意の整理）

- **WordPress 標準の「投稿（post）」**をベースに、**会員向け**に**閲覧・投稿・分類**を揃える。  
- **プラグイン量産で賄う**より、**コアに近い**実装を優先（保守説明のしやすさ・**既存環境への慎重**さ）。  
- カテゴリは **管理者が管理**（**会員の自由追加**は想定外）。

### 2.3 案件B（プラグイン整理）の方針

- **一気に削除**はしない。**棚卸し → 分類 → 重複 → 主利用4領域との関係** を明確化。  
- 各プラグインを **停止候補 / 保留候補 / 維持候補** に振り、**根拠とリスク**を文書化。  
- **「今は触らないが維持」**も**価値ある結論**として残す。

---

## 3. 案件Aで実現する体験（Who / どう使うか）

### 3.1 利用者

- **会員**: 会員エリア内で **一覧 → 詳細 → コメント**、**投稿**（**誰に許可するかは要合意**）。  
- **管理者**: **カテゴリ管理**、必要なら **投稿の承認・非公開** 等（**要否は要合意**）。  
- **非会員**: **当該エリアの閲覧不可**（既存会員仕様に準拠）。

### 3.2 主な機能（モック前の仮置き可）

- 会員限定の **閲覧・投稿**、**コメント**、**カテゴリ絞り込み**（**左サイドバー**想定）、**キーワード検索**、**投稿用フォーム**（**管理画面直ではなく**、会員向けに**分かりやすい**導線が望ましい**可能性** ※**要合意**）。

### 3.3 未確定で、本日またはモック前に欲しい合意

| 論点 | 候補 |
|------|------|
| 誰が投稿できるか | 全会員 / 役割を限定 等 |
| 承認 | 即時公開 / 管理者承認 等 |
| コメント | 有効・制限（会員のみ等） |
| カテゴリ | 初期の本数・名前、一般ブログ用と**分けるか** |

## 4. 案件Bで得られるもの

- **プラグイン一覧**（有効/無効、版）。  
- **役割分類**、**主利用4領域**（トップ・会員・掲示板・プロフィール編集）**との紐づけ**。  
- **重複候補**、**推奨（停止/保留/維持）**と**短い根拠**。  
- **追加調査**が要る項目のリスト。  
- （環境に応じ）**ステージングでの確認手順案** または **本番制約下の**留意点。

**含めない（別見積の例）**: 会員系の**大型リプレース**、**本番での大量削除**の一括代行、**作者への問い合わせ**（依頼すればオプション）。

## 5. 2案件の関係と進め方（When）

- **会員**まわりのプラグインは **案件Aの投稿・閲覧制御**に**影響**し得る。  
- 進め方の例: **B の棚卸しを先行**して依存関係を掴む / **A のモック合意**を優先 等、**環境と不安の度合い**で調整。  
- 提案書では **「推奨スケジュール案」** を **前提条件**（ステージングの有無等）**付き**で1つ示す。

---

## 6. スコープの切り方（範囲の線引き）

### 6.1 初期（最小）に寄せる例

- 標準**投稿**＋会員制限＋**カテゴリ**＋**検索**＋**コメント**（仕様は合意範囲内）。  
- **大規模データ移行**、**多段階稟議**、**添付・通知**は**初期外**扱いしやすい（**要合意**）。

### 6.2 将来拡張（今回の見積の「追加」扱いにしやすい例）

- 添付、通知、いいね、グループ別閲覧、高度な検索 等。

---

## 7. リスクとお願い（留意点）

- 既存テーマ・会員仕様の**都合**で、**想定通りの権限**に**微調整**が入る可能性。  
- プラグイン**停止**は、**本番**では**一括**より**段階的**＋**バックアップ**前提。  
- **決済・入会**系は、**合意のない削除**をしない（**B の整理**で**「触らない」範囲**を**明文化**）。
