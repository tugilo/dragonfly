# 0. 結論（先にここだけ読めばOK）

今回のご提案は、**会員向けの投稿・案件共有型掲示板**を、WordPressの標準投稿機能を活かして作るものです。  
会員が**投稿し、探し、コメントできる**状態を、まずは最小の形で整えます。

あわせて、既存のプラグインは**いきなり削除するのではなく**、役割を整理し、影響を確認しながら安全に進めます。  
費用感は、**掲示板のみで15万円**、**掲示板＋プラグイン整理で20万円**の2案です。

最初から大きく作り直すのではなく、**今あるWordPressを活かしながら、使いやすく、今後も触りやすい状態にしていく**進め方を前提にしています。

# 1. 判断ポイント（ここだけご確認ください）

今回の内容で進めるにあたり、以下の点をご判断いただければ進行可能です。

- 掲示板を**投稿・案件共有型**で進めてよいか
- 投稿を**全会員に許可するか**
- **承認フローは不要**でよいか
- **タグは初期は任意**でよいか
- **プラグイン整理を今回どこまで含めるか**（掲示板のみ / 一緒に整理）

問題なければ、この内容で**モック最終調整・見積確定**に進みます。  
必要であれば、この判断ポイントに沿って短い打ち合わせを1回入れれば、次の段階へ進めやすくなります。

# 2. 背景と課題

現在のサイトは、引き継いだWordPress環境の上で運用されており、必要な機能は残っている一方で、全体として「触るのが少し怖い」状態になっていると考えています。

特に大きい課題は次の2点です。

- 掲示板が、今後も安心して使い続けられる形になっていない  
- プラグインが多く、何がどこに効いているか分かりにくい

掲示板については、一般的なチャットのように短いやり取りを流す場ではなく、案件や情報を投稿して、あとから見返したり探したりできる形のほうが、実際の使い方に合っています。  
そのため、今の「使いにくさ」をそのまま残すのではなく、**案件共有の場として使いやすい形に整える**必要があります。

また、プラグインについては、単に「消せば軽くなる」という話ではありません。  
過去の経緯上、何かを止めたことで別の場所が壊れる可能性もあるため、まずは**役割を整理し、影響を確認しながら安全に進めること**が重要です。

---

# 3. 今回やりたいこと（結論）

今回やりたいことは、シンプルに言うと次の3つです。

- 会員が案件や情報を投稿・共有できる仕組みを作る
- 必要な投稿を探しやすく、見やすくする
- WordPress全体を、今後も触りやすい状態に少しずつ整える

今回の提案は、**大きく作り直す**方向ではなく、**今あるWordPressを活かしながら、最小スコープで使える状態にする**ことを目的にしています。

---

# 4. 解決方法（今回の提案）

## 3-1. 掲示板

掲示板は、WordPressの標準投稿機能をベースにして構築します。

方向性は、チャット型ではなく、**投稿・案件共有型**です。  
会員が投稿を読み、必要に応じてコメントし、あとから検索やカテゴリで探せるようにします。

今回の基本機能は次を想定しています。

- 投稿一覧
- 投稿詳細
- 会員による新規投稿
- コメント
- カテゴリ絞り込み
- キーワード検索

カテゴリは管理者が管理し、会員は投稿時に選択のみ行う形です。  
この形にすることで、分類が増えすぎたり、使い方が人によってばらつくことを防ぎやすくなります。

## 3-2. プラグイン整理

プラグイン整理は、いきなり削除する作業ではありません。

まずは、

- 何が入っているかを一覧にする
- それぞれの役割を分ける
- 重複や不要候補を見つける
- 今使っている画面への影響を確認する

という順番で、**安全に整理するための土台を作る**提案です。

そのうえで、

- 維持したほうがよいもの
- 保留すべきもの
- 停止候補になりそうなもの

を整理し、本番で止めるかどうかは別途判断できる状態にします。

# 5. モック説明

今回のモックは、実装コードではなく、**実際に使うときのイメージを確認するための画面案**です。

モック確認URL: [https://tugilo.com/mock/wordpress-bulletin-client-mock.html](https://tugilo.com/mock/wordpress-bulletin-client-mock.html)

## 4-1. 一覧画面

この画面では、会員が投稿を探すことができます。

- 左にカテゴリ一覧があり、見たい分類に絞れます
- 右上に検索があり、キーワードで探せます
- 中央には投稿カードが並び、タイトル・投稿者・投稿日・コメント数・画像枚数などが見えます
- 各投稿には現場イメージを複数枚持てる想定で、一覧では先頭の1枚が代表表示されます

つまり、**新しい案件を眺める**だけでなく、**必要な案件を後から探せる**画面です。

## 4-2. 詳細画面

投稿をクリックすると、詳細画面に進みます。

- 投稿の本文がしっかり読めます
- 投稿者や投稿日などの情報が確認できます
- 付いている現場イメージを複数枚見られます
- コメント欄で会員同士がやり取りできます

この画面は、単なる掲示板というより、**案件内容を確認して、必要な反応を返す場**として使うイメージです。

## 4-3. 投稿画面

会員はこの画面から新規投稿できます。

- タイトル
- 本文
- カテゴリ選択
- タグ（任意）
- 現場イメージの登録

という形で、難しい操作を増やさず、必要な情報を入力して投稿できるようにしています。

「カテゴリは管理者が管理している」「会員のみ投稿可能」といった前提も、画面上で分かるようにしています。

# 4.5 この提案のポイント

今回の進め方がよいと考えている理由は、**大きく作り直さずに、必要なところから前に進められる**ためです。

- **コストを抑えやすい**  
  既存のWordPressを活かすため、最初から大規模な作り直しにしなくて済みます。

- **進めるスピードを出しやすい**  
  モックで使うイメージを確認しながら、最小スコープで先に形にできます。

- **安全に進めやすい**  
  プラグインは一括削除ではなく整理から始めるため、別の場所を壊すリスクを抑えやすくなります。

今回の価値は、単に画面を増やすことではなく、**今ある環境を活かしながら、使いやすくしていくこと**にあります。  
そのため、「全部作り直す」よりも、現実的で進めやすい提案になっています。

# 6. 初期構築でできること

初期構築では、まず「使える状態」にすることを優先します。

会員目線でいうと、次のことができる状態を目指します。

- 会員が投稿を読む
- 会員が投稿を探す
- カテゴリで絞り込む
- キーワードで検索する
- 投稿にコメントする
- 会員が新しく投稿する
- 管理者がカテゴリを管理する

これだけでも、案件共有の場としては十分に使い始められます。  
最初から機能を増やしすぎず、必要なものを後から足せる形にするのが、今回の進め方です。

---

# 7. 今回やらないこと（重要）

今回の提案では、あえて対象外にしているものがあります。

- チャット機能
- 大規模なデータ移行
- 高度な承認フロー
- 会員システムの全面改修
- 本番での大量プラグイン停止・削除

ここを最初から全部入れてしまうと、費用も期間も読みづらくなります。  
まずは**今必要な範囲に絞って、使える形を作る**ことを優先します。

---

# 8. 進め方

進め方は、できるだけ分かりやすく段階で進めます。

1. 要件の最終確認  
   投稿権限、承認の有無、タグ利用、プラグイン整理範囲などを確認します。

2. モック確認  
   画面イメージを見ながら、「これで使えそうか」をすり合わせます。

3. 開発  
   まずは最小スコープで、会員向け掲示板を形にします。必要に応じて並行してプラグイン整理の調査も進めます。

4. 確認・調整  
   実際の表示や使い勝手を見て、軽微な調整を行います。

5. 公開  
   問題がなければ本番へ反映します。

この順番にすることで、途中で大きくズレるリスクを減らせます。

# 9. 費用感

今回は、**判断しやすいように**、次の**2つの金額案**で提案します。

## パターン①（掲示板のみ）

**15万円**

対象:

- 会員向け掲示板の初期構築
- 一覧 / 詳細 / 投稿画面
- カテゴリ絞り込み
- 検索
- コメント
- 会員限定の導線調整

まずは「掲示板を使える形にしたい」という場合に向いています。

## パターン②（掲示板＋プラグイン整理）

**20万円（おすすめ）**

対象:

- 掲示板の初期構築
- プラグインの棚卸し
- 役割分類
- 停止候補 / 維持候補 / 保留候補の整理
- 影響確認観点の整理

※ ここで含むのは、**「何を止めると危ないか」「何を残すべきか」を整理するところまで**です。  
**本番で実際にプラグインを停止・削除する作業は、この20万円には含めません。**  
実停止が必要な場合は、対象本数や確認範囲を見たうえで**別見積**とします。

今後も継続してサイトを触ることを考えると、**掲示板だけ直して終わるより、全体の整理も一緒に進めたほうが安心感が高い**ため、こちらをおすすめします。

# 10. なぜこの金額になるのか

金額は、単に作業時間ではなく、**安心して進めるために必要な工程**を含めて考えています。

画面を作るだけでなく、方向性の確認、既存環境への合わせ込み、公開前の確認まで含めて進めるため、この金額感になります。  
「とにかく安く作る」よりも、**無理なく進められて、あとで困りにくい形にする**ことを重視しています。

具体的には、

- 方向性を決める
- 画面を確認する
- WordPress上で無理のない形に落とす
- 既存環境に合わせて壊れにくく調整する
- 公開前に確認する
- プラグイン整理では、**削除前に影響を見える化する**

という流れが必要です。

特に今回の案件は、完全な新規開発ではなく、**既存環境に合わせながら進める**ことに価値があります。  
そのため、安さだけで進めるより、**今後も触れる状態に整えること**を含めた金額として考えていただくのが適切です。  
なお、**プラグインの本番停止・削除は、調査の延長でそのまま一括実施するものではない**ため、必要な場合は切り分けて判断する前提です。
