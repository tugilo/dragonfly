## 1. はじめに

国宝社様の製本販売管理は、三菱関連会社製 **VB + Oracle** の基幹システムが長年の中心として機能しています。本見積は、その基幹を **作り替えず**、**PDF注文書の手入力負担** を減らす **周辺Webシステム（入力支援）** の投資規模の目安です。

第2回121（2026-05-29）で合意したとおり:

- **Google Workspace / GAS 案は不採用**
- **社内ローカルサーバー（Hyper-V 上 Linux Web）** で展開
- **PDF解析 → 人が確認・修正 → Oracle 登録**（100%自動化は目指さない）

---

## 2. 見積の前提

| 前提 | 内容 |
|------|------|
| 対象業務 | 出版社・大口得意先から届く **PC作成 PDF 注文書** の受注入力 |
| 対象件数感 | 大口顧客 **月300〜400件**（全体は倍近い可能性） |
| 既存システム | VB + Oracle **温存**。テーブル構造変更なし |
| データ連携 | 受注ヘッダー **TTJUCUHD** / 明細 **TTJUCUDT** への書き込み |
| インフラ | 社内 Hyper-V 上 Linux Web。外部公開なし。VPN 経由アクセス |
| 初期フォーマット | **1〜2出版社パターン** から開始（追加は段階的） |
| 含まない | 手書きOCR、基幹リプレイス、得意先ポータル、外部クラウド |

## 3. 概算の考え方（3段階）

いきなり全フォーマット・全件を対象にせず、**効果とリスクを確認しながら段階投入** する前提で、3段階に分けています。

```text
Step 0: 現状整理・技術検証（PoC）
  ↓
Phase 1-A: 入力支援Web（初期1フォーマット・限定運用）
  ↓
Phase 1-B: フォーマット追加・本番運用拡大
  ↓
Phase 2: 日報・集計のWeb化 等（別途）
```

## 9. 投資対効果（ROI）— 人件費ベース試算

### 9.1 なぜ人件費ベースか

本案件の Phase 1 は、**事務員の PDF → 手入力** という **人の時間** が主コストです。  
したがって ROI は次の式で説明するのが自然です。

```text
年間削減効果（円）＝ 削減時間（時間/年）× 人件費単価（円/時間）
投資回収期間（年）  ＝ 初期投資（円）÷ 年間削減効果（円）
```

**メリット（見積・導入判断）**

- 「180万円」だけより **何年で元が取れるか** が伝わり、経営判断しやすい
- 木村さんに **1件何分か** を確認する理由が明確になる
- 削減率を保守/標準/楽観で示せば、**見積の信憑性** が上がる（断定を避けつつレンジで示す）

**注意（人件費だけにしない理由）**

| 含めやすい | 含めにくいが重要 |
|------------|------------------|
| 入力作業時間の削減 | 入力ミスによる **差し戻し・手戻り** |
| 確認作業へのシフト（部分） | 繁忙期の **残業・派遣** |
| 担当者の **確認・修正** 時間（解析後） | 教育・引き継ぎコストの削減 |
| | 機会損失（入力待ちで受注処理が遅れる） |

→ スライド・見積では **「直接の入力作業時間」** を主軸にし、ミス低減は **定性のプラスα** として補足するのがよい。

---

### 9.2 試算に必要な確認事項（木村さん側）

| 項目 | 用途 | 状態 |
|------|------|------|
| 1件あたりの入力時間（平均・最大） | 削減時間の算出 | **要確認** |
| 対象件数（月・大口のみ / 全体） | 削減時間の算出 | 大口350件/月は121より |
| 担当人数 | 単価の掛け方 | **要確認** |
| 人件費単価（時給換算 or 社内目安） | 金額換算 | **要確認**（不明時はレンジ試算） |
| 繁忙期のピーク | 投資優先度の説明 | 任意 |

**人件費単価の扱い**

- **理想:** 国宝社様の社内目安（事務担当の時間単価・社会保険込み等）
- **不明時:** 下表の **参考レンジ** で3パターン試算し、「単価確定後に再計算」と明記

| 参考（_loaded cost・事務系） | 円/時間 | 備考 |
|------------------------------|---------|------|
| 保守 | ¥2,000 | 社内按算の下限イメージ |
| 標準 | ¥2,500 | 中小製造・事務の中間イメージ |
| 上限参考 | ¥3,000 | 社保・間接費を厚めに見た場合 |

※上記は **試算用の参考値** であり、国宝社様の実数ではない。

### 9.3 試算モデル（3シナリオ）

**共通前提（121・受領資料ベース）**

- 対象: 大口顧客 PDF **月350件**
- Phase 1 対象: PC作成PDF・初期1〜2フォーマット
- 削減の意味: **純粋な手入力時間** の短縮（確認・修正時間は別途発生）

| シナリオ | 1件入力時間 | 削減率 | 根拠イメージ |
|----------|-------------|--------|--------------|
| **保守** | 10分/件 | 30% | 確認・修正が多く、初期フォーマット1つのみ |
| **標準** | 15分/件 | 40% | 解析＋確認で転記中心を削減 |
| **楽観** | 20分/件 | 50% | 1件あたり負担が大きく、フォーマット適合が良い |

**削減時間（月・1名分の作業換算）**

| シナリオ | 計算 | 月間削減 | 年間削減 |
|----------|------|----------|----------|
| 保守 | 350 × 10分 × 30% | **17.5時間** | **210時間** |
| 標準 | 350 × 15分 × 40% | **35時間** | **420時間** |
| 楽観 | 350 × 20分 × 50% | **58時間** | **700時間** |

---

### 9.4 年間削減効果・回収期間（パッケージ B = 180万円想定）

**年間削減効果 = 年間削減時間 × 人件費単価（¥2,500/時間 で統一した例）**

| シナリオ | 年間削減時間 | 年間効果（@2,500円/h） | 回収期間（180万÷年間効果） |
|----------|--------------|------------------------|----------------------------|
| 保守 | 210時間 | **¥525,000** | **約3.4年** |
| 標準 | 420時間 | **¥1,050,000** | **約1.7年** |
| 楽観 | 700時間 | **¥1,750,000** | **約1.0年** |

**人件費単価を変えた場合（標準シナリオ・420時間/年）**

| 単価 | 年間効果 | 回収期間 |
|------|----------|----------|
| ¥2,000/h | ¥840,000 | 約2.1年 |
| ¥2,500/h | ¥1,050,000 | 約1.7年 |
| ¥3,000/h | ¥1,260,000 | 約1.4年 |

---

### 9.5 導入判断の読み方（提案時の伝え方）

| 回収期間 | 判断の目安 |
|----------|------------|
| **〜2年** | 事務負担・ミスリスクを考えると **導入検討しやすい** |
| **2〜3年** | **段階導入（検証パックA→B）** でリスクを抑えつつ判断 |
| **3年超** | 1件時間・件数・削減率の **再確認**、または Phase 1 スコープの見直し |

**パッケージ A（検証・約50万）の位置づけ**

- 180万を一括判断する前に、**50万で「本当に40%近く削れるか」** を確認できる
- ROI 試算の **最大の不確実要素**（1件時間・削減率）を Step 0 で潰せる → **見積全体の信憑性が上がる**
