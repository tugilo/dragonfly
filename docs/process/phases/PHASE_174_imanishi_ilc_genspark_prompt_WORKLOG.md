# PHASE_174_imanishi_ilc_genspark_prompt WORKLOG

## 判断ログ

### プロンプトの役割

Gensparkでスライドを作る際に、単に提案書本文を貼るだけだと、金額や技術要素に寄りすぎる可能性がある。今回の資料は経営者向けに「手間削減・費用対効果・段階提案」を伝える必要があるため、スライド構成まで指定するプロンプトにした。

### スライド構成

10〜12枚程度とし、結論、現状課題、検証結果、削減効果、ROI、提案プラン、推奨ステップ、AI駆動開発、次確認事項の順にした。

### 表現方針

「AIで全部自動化できます」と断定せず、見込み・仮説・検証前提を明記するようにした。売り込み感を抑え、まず小さく検証する提案に寄せた。
