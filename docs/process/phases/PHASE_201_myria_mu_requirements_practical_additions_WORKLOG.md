# PHASE_201_myria_mu_requirements_practical_additions WORKLOG

## Task1 - 施工店ステータスと対応エリア
- 状態: completed
- 判断: 施工店が増えた場合、送客対象外・契約終了・一時停止が発生するため、施工店本体にステータスを持たせる必要がある。
- 実施: 施工店ステータス（審査中 / 有効 / 一時停止 / 解約）を追加し、有効な施工店のみ候補表示対象とする要件を追記した。
- 確認: 対応エリアは `contractor_service_areas` 相当の別データとして複数登録できる前提に変更した。

## Task2 - 案件履歴
- 状態: completed
- 判断: 案件ステータス変更や支払状況変更では、後から「誰がいつ変更したか」を確認できることが必要。
- 実施: 案件履歴要件を追加し、変更前/変更後ステータス、変更種別、変更者、変更日時、備考をデータ要件に追加した。
- 確認: 案件ステータス変更、担当施工店変更、契約登録、完了報告、報酬確定、支払状況変更、イラスト制作ステータス変更を記録対象にした。

## Task3 - 施工店評価指標
- 状態: completed
- 判断: Myria-mu の価値は良い施工店を紹介できることにあるため、初期から評価に使えるデータ構造を用意する。
- 実施: 施工店評価指標として、送客件数、契約件数、契約率、平均満足度、平均NPS相当スコアを追加した。
- 確認: 高度な評価ダッシュボードは初期スコープ外としつつ、データ構造は初期スコープに含めた。

## Task4 - イラスト制作管理
- 状態: completed
- 判断: Myria-mu 独自の完了後業務として、アンケート・写真回収後のイラスト制作進捗を案件単位で管理する必要がある。
- 実施: `illustration_status`（未依頼 / 制作待ち / 制作中 / 送付済）を案件データに追加し、機能要件とスコープに反映した。
- 確認: 外部制作ツール連携は初期対象外とし、ステータス変更は案件履歴に記録する要件とした。

## Task5 - NPS相当アンケート項目
- 状態: completed
- 判断: 顧客満足度だけでなく「施工店を知人に勧めたいか」は施工店評価に使える重要指標である。
- 実施: 顧客アンケートに `recommendation_score` を追加し、施工店評価指標に利用する要件を追記した。
- 確認: NPS相当項目の形式（0〜10点か段階式か）は確認事項に残した。
