# PHASE_167_imanishi_ilc_webapp_cost WORKLOG

## 判断ログ

### Web アプリ前提への変更

ユーザー方針に合わせ、デスクトップアプリではなく Web アプリとして実装する費用感に更新した。

Web アプリ前提にする理由は、担当者PCの環境差に依存しにくいこと、複数人で使いやすいこと、将来的にログイン・取込履歴・送信履歴・宛先管理へ拡張しやすいこと。

### 費用レンジの見直し

Phase 166 の費用感は、スクリプトや簡易ツールに近い粒度だった。Web アプリでは画面、アップロード処理、サーバー側集計、ファイル出力、認証、履歴管理などの要素が入るため、以下のようにレンジを見直した。

- Web検証: 8〜15万円
- Webアップロード最小版: 45〜80万円
- 加盟店向けWebレポート生成版: 80〜140万円
- Web管理画面付き運用版: 120〜200万円
- 送信管理込みWeb運用版: 180〜300万円

### ランニング費用

Web アプリでは初期構築費だけでなく、サーバー・ドメイン・保守が必要になる。そのため月額の目安も相談メモに追記した。
