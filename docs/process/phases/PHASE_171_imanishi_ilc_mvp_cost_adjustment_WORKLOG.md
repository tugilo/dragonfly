# PHASE_171_imanishi_ilc_mvp_cost_adjustment WORKLOG

## 判断ログ

### 費用感の調整

Phase 168 で安全側に費用を上げたが、今西さんへの初回提示としては高く見えすぎる可能性がある。ユーザーの「高いかな」という指摘を受け、初回提案はフル機能ではなく MVP 中心にする方針にした。

### MVP の定義

MVP では、ブラウザから TSV をアップロードし、店舗別・日別・項目別に Excel / CSV を出力する範囲に絞る。ログイン、送信管理、高度な履歴管理は初期スコープから外す。

これにより費用感を 80〜90万円程度に下げ、投資回収期間も短く見せられる。

### フル機能の扱い

送信管理や詳細な履歴管理は、初回提案では前面に出さず将来拡張として整理した。これにより、初期検討に入りやすくしつつ、将来的な拡張余地も残した。
