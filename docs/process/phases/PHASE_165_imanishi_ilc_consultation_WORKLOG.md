# PHASE_165_imanishi_ilc_consultation WORKLOG

## 判断ログ

### 相談メモの位置づけ

今西さんの相談は、正式な提案書ではなく「既存データと運用を見て、何ができそうか検証してほしい」という段階である。そのため `docs/proposals/` ではなく、Phase 164 で新設した `docs/consultations/` の相談メモとして扱うことにした。

### AiLC 配下の README

`docs/consultations/ILC/` には TSV・Excel・PDF が多数入っており、相談メモ単体ではデータの位置づけが分かりにくい。今後、検証メモや試作結果が増える可能性があるため、AiLC 配下に README を置き、相談メモとデータフォルダの役割を整理した。

### 初期検証の範囲

本 Phase は docs フェーズのため、集計ツール実装や成果物生成までは行わない。一方で、相談メモとして実用的にするには「自動化できそうか」の判断材料が必要なため、以下のみ確認した。

- 手順 PDF の作業概要
- TSV の文字コード・列構造
- チカラグループ / ファボローネの TSV 件数
- 代表的な Excel のシート構成

### 技術的な初期判断

TSV は `cp932` で読み取り可能で、列構造も共通している。日付・店舗・category/action/label・件数が取れるため、日別・店舗別・項目別の集計はスクリプト化できる可能性が高い。

既存 Excel / xlsm はマクロ・数式・手作業転記に依存しているため、完全再現よりも「TSV を正として新しい見せ方で出す」方が安全と判断した。相談者からも見せ方変更は問題ないとの情報がある。
