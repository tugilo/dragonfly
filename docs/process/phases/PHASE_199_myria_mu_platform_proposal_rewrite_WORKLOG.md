# PHASE_199_myria_mu_platform_proposal_rewrite WORKLOG

## Task1 - 提案視点の変更
- 状態: completed
- 判断: 既存提案書は要件整理としては成立していたが、「管理システム導入」の印象が強かったため、事業成長を支える外構紹介プラットフォームの提案へ軸を移す。
- 実施: タイトルを「Myria-mu 外構紹介プラットフォーム構築提案」へ変更し、サブタイトル「インフルエンサー・施工店・顧客をつなぐ外構紹介ネットワークの構築」を追加した。
- 確認: 冒頭で、構築すべきものは顧客管理システムではなく外構紹介プラットフォームであると明記した。

## Task2 - 提案の要旨と未来像
- 状態: completed
- 判断: 管理負荷軽減は副次効果とし、Myria-mu 独自の紹介ネットワーク、インフルエンサー集客、施工店パートナー、顧客満足、施工事例、データ活用を主軸に置く。
- 実施: 提案の要旨を全面修正し、Mermaid 図をインフルエンサー→顧客→Myria-mu→施工店パートナー→顧客満足→施工事例→データ蓄積→次の施策の循環に拡張した。
- 確認: 最終的に「紹介ネットワークが成長する循環」として表現した。

## Task3 - パートナー定義の強化
- 状態: completed
- 判断: 施工店は送客先ではなく地域パートナー、インフルエンサーは報酬管理対象ではなく成果パートナーとして表現する。
- 実施: インフルエンサー成果指標（問い合わせ件数、契約件数、契約金額、報酬額）と、施工店パートナー評価指標（対応品質、契約率、顧客満足度等）を追記した。
- 確認: Myria-mu と各パートナーの長期関係を育てる仕組みとして記載した。

## Task4 - 顧客満足と施工事例資産化
- 状態: completed
- 判断: 顧客アンケートは管理項目ではなく、次の集客を生む営業資産として扱う。
- 実施: 工事完了→顧客アンケート→完成写真回収→オリジナルイラスト制作→顧客へプレゼント→SNS投稿・施工事例活用→次の集客の流れを追加した。
- 確認: 新規章「施工事例資産化」を追加し、施工事例集、SNS投稿素材、施工店紹介ページ、営業資料への将来活用を記載した。

## Task5 - 専用システム化と予算説明
- 状態: completed
- 判断: スプレッドシートとの違いを、管理台帳ではなく複雑な関係性を扱うP-LINK型プラットフォームとして説明する。
- 実施: 新規章「なぜスプレッドシートではなく専用システムなのか」を追加し、150万〜180万円を外構紹介プラットフォーム基盤構築費として説明した。
- 確認: 初期費用を単なる業務効率化費ではなく、将来の紹介ネットワーク拡大への投資として表現した。
