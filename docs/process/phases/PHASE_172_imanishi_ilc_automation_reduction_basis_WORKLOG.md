# PHASE_172_imanishi_ilc_automation_reduction_basis WORKLOG

## 判断ログ

### 削減率の見直し

既存の ROI 表は削減率がやや保守的だった。ユーザーの指摘どおり、TSV取得後の加工〜集計表作成に絞れば、もっと削減できる可能性が高い。

手元データでは、チカラグループ 15 TSV、ファボローネ 36 TSV、合計 51 TSV を確認した。データ行は合計 76,628 行で、列構造は共通している。

この処理は、店舗ごとの TSV を読み込み、日付・category・action・label・件数で集計し、欠損日を補完して Excel / CSV に出力する定型処理である。そのため、手作業の貼り付け・変換・転記部分は 80〜95% 削減を見込めると判断した。

### 範囲の分離

削減率を正しく伝えるため、以下を分けた。

- TSV取得後の加工〜集計表作成
- CMSからのTSV取得・送付確認まで含む月次運用全体

前者は大きく削減できるが、後者には人の確認・送付・承認作業が残るため削減率は少し下がる。
