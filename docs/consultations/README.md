# メンバー相談メモ（docs/consultations/）

**用途:** メンバーから受けた細かい相談事を、調査・検討・次アクションに落とし込むための置き場。  
1to1 の関係履歴とは分けて、「この相談に対して何ができるか」を考える作業ログとして管理する。

---

## 置くもの

| 種別 | 例 |
|------|----|
| 業務改善の小さな相談 | 予約管理、日報、手入力削減、集客導線、顧客管理など |
| 相談への調査メモ | 既存業務の整理、類似事例、使えそうなツール、実装難度 |
| 対応方針の検討 | 無償で返せる助言、簡易モック、提案書化、他メンバー紹介 |
| 次アクション | 追加ヒアリング、資料作成、見積前の確認事項、紹介候補 |

---

## 置かないもの

| 種別 | 置き場 |
|------|--------|
| 1to1 の関係履歴・議事録 | `docs/meetings/1to1/` |
| クライアント／パートナーに共有する提案書・ブリーフ | `docs/proposals/` |
| Religo の仕様として確定した要件 | `docs/SSOT/` または `docs/02_specifications/` |
| 実装タスクの Phase 記録 | `docs/process/phases/` |

相談メモから提案書に発展した場合も、元の相談メモは残し、提案書側から参照する。

---

## ファイル命名

原則は次の形式にする。

```text
consultation_<person_slug>_<topic_slug>.md
```

例:

```text
consultation_nishioka_line_daily_report.md
consultation_funatsu_reservation_management.md
consultation_kimura_pdf_order_entry.md
```

判断に迷う場合は、後で探しやすいように「相談者」と「相談テーマ」を優先する。日付は本文に記録し、ファイル名には原則入れない。

---

## 推奨フォーマット

```markdown
# 相談: <相談者> / <テーマ>

## 基本情報
- 相談者:
- 所属・会社:
- 初回相談日時: YYYY-MM-DD HH:MM JST
- 相談の出所: 1to1 / 定例会 / チャット / 紹介 / その他
- 関連1to1:

## 相談内容
- 困っていること:
- 現状の運用:
- 相談者が期待していること:
- 制約・前提:

## 調査・検討
- できそうなこと:
- すぐ返せる助言:
- 追加確認が必要なこと:
- リスク・注意点:

## 対応案
| 案 | 内容 | 工数感 | 費用感 | メモ |
|----|------|--------|--------|------|
| A | | | | |
| B | | | | |

## 次アクション
- [ ] 

## 関連リンク
- 1to1:
- 提案書:
- 参考資料:
```

---

## 運用ルール

1. 相談を受けたら、まず事実と未確定事項を分けて書く。
2. 相談者に確認していない推測は、断定せず `仮説` または `TODO` として残す。
3. 調査結果は「できること」「やらないこと」「追加確認が必要なこと」に分ける。
4. 提案書化できる粒度になったら `docs/proposals/` に別ファイルを作り、相談メモからリンクする。
5. 1to1 の履歴に関わる内容は、該当する `docs/meetings/1to1/` のファイルにも要点を反映する。
6. 日時を記録する場合は、日付だけでなく時刻と `JST` を書く。不明な場合は `TODO` と明記する。
