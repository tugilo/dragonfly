## 使い方

- 講義中は、まず講師の言葉・スライド番号・ワークの結果をそのまま残す。
- 各節の見出しに **スライド番号（配布 p.x）** を付けてある。当日の進行と違う場合は **メモ側を正** とする。
- 「メモ（議事録）」欄に聞き取りを書き、「自分・DragonFly メモ」欄に自チャプターへの当てはめを書く。
- パーキングロットに、後で調べたい・1to1で聞きたい項目を随時追記。
- **このプログラムの主役は「メンターコーディネーター（メンターC）＋10種類のメンター」** で新メンバーを 8〜12週で立ち上げる仕組み。自分が **どの役割を担えるか／DragonFly でどう運用するか** を各メンターのところでメモする。
- 各節の **「台本（読み上げ用）」** は、ブレイクアウト・振り返り・将来メンター役を担うときの原稿。出典: [Living Document §8](../../strategy/networking/BNI_Tsugihiro_Atsushi_Intro_Living_Document.md) をベースに tugilo／DragonFly 向けに調整。

## 4. 1年目メンバーの更新率（最大の課題）

**スライド:** 5（配布 p.2）
**記録時刻:**

### スライド要点

- **BNI メンバーを続けるのが最も難しいのは「1年目」**
- 1年目メンバーの更新率 → **約 53.0%**
- 全メンバーの更新率 → **約 64.0%**（※2025年8月現在）

### 台本（読み上げ用）

```
BNI でメンバーを続けるのがいちばん難しいのは「1年目」。
1年目の更新率は約53%、全体でも約64%——半数近くが1年で離れていく。

チャプターにとっては、せっかく迎えた新メンバーが静かに去ると、
またゼロから信頼関係を築き直すコストがかかる。
だからパスポートプログラムは「1年目を支える」仕組みだと理解しています。
```

## 5. 更新率とメンバー数の関係

**スライド:** 6–7（配布 p.2–3）
**記録時刻:**

### スライド要点（メンバー数30・入会者数別シミュレーション）

- **月1名入会**（slide 6）：更新率70% → 1年後 29.4 / 2年後 28.9 / 3年後 28.6。更新率40% → 16.8 / 11.5 / 9.4（＝先細り）
- **月2名入会**（slide 7）：更新率70% → 1年後 37.8 / 2年後 43.2 / 3年後 **47.0**（＝右肩上がり）。更新率40% → 21.6 / 18.2 / 16.8
- **示唆:** 入会数だけでなく **更新率を上げないとチャプターは増えない**。パスポート＝1年目の更新率を上げる仕組み。

### 台本（読み上げ用）

```
入会者を増やすだけではチャプターは伸びない——更新率が鍵です。

月1名入会で更新率40%だと、3年後はメンバー数が9名台まで落ちる。
月2名入会で更新率70%なら、3年後は47名まで伸びる。

DragonFly でも、BODで人を迎えるだけでなく、
1年目メンバーが「恩恵を実感して更新する」仕組みが必要だと感じています。
```

# C. メンバージャーニーと価値

## 8. 1年目の BNI メンバージャーニー（学習期間）

**スライド:** 10（配布 p.4）

### スライド要点（3ステップ）

| 期間 | フェーズ | 内容 |

4〜6ヶ月：信頼をビジネス関係に。ギバーズゲインを実践する準備。
7〜12ヶ月：信頼が固まり、質の高いリファーラルが増える。

最大のベネフィットは売上だけでなく、一生続く信頼関係——
パスポートはその「最初の12週」を支える設計だと理解しています。

## 10. メンバー歴が長いほど RF の価値が大きい

**スライド:** 12–13（配布 p.4–5）
**記録時刻:**

### スライド要点（活動配分の対比）

| チャプターの状態 | メンバーを増やす活動 | RF を交わす活動 |
|------------------|----------------------|-----------------|
| 経験豊富45人で構成（理想） | **20%** | **80%** |
| メンバーが長く留まらない（課題） | **80%** | **20%** |

- **示唆:** 定着すれば「増やす」より「交わす」に時間を使える。定着しないと永遠に「増やす」に追われる → パスポートで定着を促す。

### 台本（読み上げ用）

```
経験豊富なチャプターは、活動の80%をリファーラル交換に使える。
定着しないチャプターは80%を「メンバー募集」に使い続ける。

定着すれば、みんなが紹介に集中できる。
パスポートで1年目を支えるのは、チャプター全体の時間の使い方を変えることです。
```

## Zoom文字起こし要約（当日議事録）

**反映日時:** 2026-06-29 17:40 JST  
**ソース:** Zoom 文字起こし要約（ユーザー共有）

### 全体サマリー

BNI パスポートプログラムのワークショップとして、新メンバーの **定着率・更新率向上** をテーマに約2時間半のセッションが実施された。前半では、更新率とチャプター規模の相関データを分析し、退会の根本原因が **早期の気づきと関係構築の欠如** にあることが示された。

後半では、メンバートラフィックライトレポートの活用、ビジター招待の意義、そしてメンターの本質は **「お世話係」ではなく「次期リーダーを育てる存在」** である、という核心メッセージが共有された。ブレイクアウトルームでの実践ワークを通じて、具体的なサポート手法と関係構築の重要性を体験的に学んだ。

### 決定事項・重要な認識

- **更新率50%未満** では、毎月1名入会しても1年後にチャプターは縮小する。
- **更新率70%かつ毎月2名入会** の場合に純増が実現しやすい。更新率と新規入会の両方を同時に見る必要がある。
- 退会理由の本音（成果につながらない・時間がかかる・本業が忙しい）が出てくる時点では手遅れ。**早期発見・早期介入** が唯一の解決策。
- トピックメンターの役割は **トピックの説明ではなく関係構築**。説明係として機能しているメンターは役割の本質を見直す。
- メンターは **次期リーダーを育てる存在**。単なるお世話係・事務連絡係ではない。
- パスポートプログラムは、40年近い BNI の研究の中で世界的に導入された仕組みであり、定着率・更新率向上に効果があるとして日本でも採用されている。

### 更新率データと現状分析

**数値で見る更新率の重要性**

- 架空の30名チャプターで毎月1名入会しても、更新率70%でも1年後はメンバー減少。更新率60%でも同様に減少し、50%未満では1年後の存続が危うい。
- 毎月2名入会の場合でも、更新率50%以下であれば純減が続く。
- アバンダンスチャプター・イーグルチャプターは、1年目更新率が安定して高い。これは **文化として定着している** と評価され、他チャプターのメンバーとの1to1が推奨された。
- 各チャプターの1年目更新率データはチャットで共有済み。DragonFly の1年目更新率も別途確認する。

**自分メモ**

- DragonFly でも BOD やビジター招待で「入れる」ことだけに寄らず、入会後12週間・1年目の定着に目を向ける。
- 高更新率チャプター（アバンダンス／イーグル）の運用を1to1で聞く。

### 退会の根本原因と早期介入

**よく聞かれる退会理由**

- ビジネス拡大できなかった／人脈構築できなかった／自分には向いていなかった／成果につながらなかった。
- 最近は「BNIで仕事が増えて忙しくなったからやめる」という理由も増えている。

**本質的な問題**

- 退会理由が出てくる前に **誰も気づいていない** のが最大の課題。
- BNIを活用できていない、関係構築ができていない、仕組みを理解できていない状態を放置している。
- 退会申し出の60日前・30日前に気づいても手遅れ。
- プログラム利用料の本質（リファーラルマーケティングを学び実践するプログラム）を理解しないまま「成果が出ない」と感じるケースが多い。
- 1年目退会者のトレーニングレポートを見ると「ホラー」と言える状態のことが多い。

**解決策**

- チャプターへの馴染み、活用状況、メンバーとの関係構築状況をできるだけ早く把握する。
- 新メンバーだけでなく既存メンバーにも継続的に状況把握を行う。
- 事務連絡ではなく、相手の現状・気持ちを聞く **ケアコール** を増やす。

### 1年目メンバージャーニーとサポート

- **1〜3ヶ月目:** BNIについて学び、信頼関係を育む期間。3ヶ月目終了時点で信頼関係が構築されていないとビジネスにつながらない。
- **4〜6ヶ月目:** 築いた信頼関係を強力なビジネス関係に発展させる期間。
- 悪い例: 「リファーラル出せてますか？」「1to1組んでますか？」「トレーニング予約しましたか？」という一方的な確認連絡。これはコミュニケーションでも関係構築でもない。
- 正しいアプローチ: 相手の現状や気持ちを聞いた上で、「だったらこういうふうにしたらどうですか？」と提案する。
- コミュニケーションは **量だけでなく質（中身）** が重要。新メンバーが関係構築のやり方自体を知らない場合もあるため、その方法も教える。

### メンバートラフィックライトレポートの活用

**レポートの本質**

- 誰がどれくらいチャプターに貢献できているかの可視化ツール。
- 困っているメンバーを発見するツール。
- BNIをどのくらい活用できているかの評価シート。
- 「伸びしろがわかる表」（シルビスチャプター・遠藤氏の表現）。
- ギバーズゲインの理念に基づき、どれくらいギブできているかの可視化。

**具体的な読み方**

| 状況 | 仮説・見るポイント |
|------|------------------|
| 内部リファーラルが多く外部リファーラルが少ない | 1to1が不足している、または自腹を切るような紹介になっている可能性 |
| 1to1件数が半年で一桁（月1〜2人程度） | 関係構築の基準を満たしていない可能性 |
| 新メンバーで CEU/CU がゼロ | Connect での CU 入力方法が分からない可能性 |

- 仮説を立てたら積極的に声をかける。
- 7ヶ月レビュー面談時には必ずこのレポートを確認する。
- 2024年4月より「講演者」欄が新設。入会1年目以内のメンバーが講演（ビジター招待）を行うことは更新率向上に相関するデータがある。
- 指摘のためではなく、成果を出すためのサポート内容を把握するために使用する。

### ビジター招待の意義

**なぜビジター招待をするのか**

- 自分の人脈をメンバーに提供する行為。
- 紹介ラインが増える。入会すれば人脈が拡大する。
- リファーラルの最高峰。直接お連れして、複数の人に一度に紹介できる場。
- 1人の新メンバーが約250人の人脈をチャプターにもたらす可能性がある。

**更新率との相関**

- 入会1年目以内のメンバーが講演（ビジター招待）を行うと、更新する確率が高くなるというデータがある。
- 自分が誘った人が入会する体験が、そのメンバー自身の定着意欲を高める。
- Netflix の例: 視聴者は2〜3話目で見続けるかを決める。BNI も同様に、早期の良い体験が更新意思につながる。

**成功のためのサポート**

- ビジターに参加する理由を作ってあげる。
- 新しい経営者と会い続ける習慣を作る。
- パワーチームになってほしい人を誘う視点を持つ。
- 相手のビジネスを把握した上で招待する。
- 成功例・失敗例を多く聞き、自分の引き出しを増やす。

### 良いメンターの条件（参加者シェア）

| シェア元 | 主な意見 |
|----------|----------|
| スプレッドチャプター・郷田氏のルーム | メンバー全員と関係構築ができる人／情報が入ってくるアンテナが立っている人／教えるだけでなく考えることを投げかけられる人 |
| デフォルトチャプター・小川原氏のルーム | 連絡しやすい・相談しやすい人／手本になる貢献度が高い人／頑張りを見逃さずに見てくれる人 |

**核心メッセージ**

- メンターは **次期リーダーを育てる存在**。
- トピックの説明係ではなく、関係構築を通じて新メンバーの成長を支援する。
- DragonFly での実感とも一致: 信頼できて相談しやすい人がメンターに選ばれる傾向がある。

### 新しい取り組み: ケアコール

- 2024年4〜5月頃から導入検討が始まった新しいアイデア。
- 現状の問題: 新メンバーへの連絡がフォーム入力依頼・アップロード依頼など、事務連絡ばかりになりがち。
- ケアコールの目的: 「最近どうですか？」「今日のプレゼン工夫されましたね？」など、相手への関心を示す連絡を意識的に行う。
- 多くの人が気にかけてくれていると感じることが、更新率・定着率向上に最も貢献する。

### パスポートプログラム運用上の注意

- トピックメンターには、トピックを説明できる人ではなく **関係構築ができるメンバー** を配置する。
- パーソナルメンターは、新メンバーと最も密に関係構築できる立場。新メンバー1人が持つ約250人の人脈に最初にアクセスできるという視点を持つ。
- トピックの順番は順不同で可。必ず1番から順番にやる必要はない。
- 新メンバーに関わる全員が **一貫したメッセージ** を持つことが必須。「Aさんはこう言った、Bさんはこう言った」という矛盾が混乱を招く。
- 参考資料: 運営マニュアル110ページ（メンター詳細）、114ページ以降（パスポートの使い方、メンターC・パーソナルメンター・トピックメンター）、TLT の最終セクション。
