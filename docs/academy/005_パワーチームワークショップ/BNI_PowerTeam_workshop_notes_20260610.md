### 原稿 2 — ターゲットマーケット（ver3.1 **p.2–5**）

#### 2-0 なぜターゲットマーケットか（p.2–3）— **先にここ**

**ターゲットマーケットとは何か（p.2）**

```
ターゲットマーケットとは、
**いちばん力になれる理想のクライアント像** を、
属性（業種・規模・役職）と **困りごと（症状）** で言語化したものです。

「誰にでも売れる」ではなく、
**誰に、どんな状態のとき、どう役立つか** をはっきりさせる作業です。
```

**なぜそれが重要なのか（ver3.1 **p.3** — スライド3項目）**

スライド表記どおり：

1. **具体的なメッセージの作成に役立つ**
2. **コストパフォーマンスに優れた戦略の構築**（※「先着」ではなく **戦略**）
3. **競争力の向上**

---

**① 具体的なメッセージの作成に役立つ**

| 項目 | 内容 |
|------|------|
| **スライドの意味** | TM が決まると、広告・紹介・プレゼンで **同じ言葉** を繰り返せる |
| **TM 前（曖昧）** | 「AIで業務改善できます」「システム作ります」→ メンバーは覚えられない |
| **TM 後（具体）** | 「**Excel限界でボヤいている経営者・管理者** を繋いでください」→ 25秒・121・PT で **同じボヤキ** |
| **自分の例** | 4つの症状ローテ（属人化／後回し／二重入力／Excel限界）。商品名より **口にした困りごと** |

**メモ用1行:** メッセージ ＝ カテゴリ名ではなく **「誰の・どんな症状」**

---

**② コストパフォーマンスに優れた戦略の構築**

| 項目 | 内容 |
|------|------|
| **スライドの意味** | 全員にアプローチせず、**当たる確率が高い層** に時間・紹介・関係構築を集中できる |
| **TM 前（非効率）** | AI興味の人・現場担当のみの1to1 → 決裁に届かず **時間が溶ける** |
| **TM 後（集中）** | **10〜50名・現場系・経営者や管理者** ＋ 士業の **金のガチョウ** ルートだけを追う |
| **自分の例** | 90分診断・PoC は **症状がはっきりした人** から。BOD も「角度の高い事業オーナー」優先（スリーバイス方針と一致） |

**メモ用1行:** CP ＝ **広く取らない** ＝ 1to1・診断・PT の **当たり率** を上げる

---

**③ 競争力の向上**

| 項目 | 内容 |
|------|------|
| **スライドの意味** | 同じカテゴリー競争ではなく、**「その人の課題ならあなた」** と選ばれる |
| **TM 前（埋もれる）** | 「AI」「システム開発」は競合・類似が多い。価格・機能比較になりやすい |
| **TM 後（選ばれる）** | 「**売上伸びなのに現場がパンク** → 構造を整えて時間を生む」—— **症状×伴走** で一本化 |
| **自分の例** | 小中さん＝AI **入口・学び**／次廣＝現場 **構造・時間・伴走開発**。DragonFly 内でも **被らない** |

**メモ用1行:** 競争力 ＝ 安さ・機能より **「その症状の専門家」** として記憶される

---

**3つを自分ごとに1文ずつ（スライド追いかけ用）**

```
① TM があると、ウィークリーで毎週 **同じボヤキ** を言える。
② 全員に会うのではなく、 **経営者・管理者と士業ルート** に時間を集中できる。
③ 「AIができる人」ではなく、 **Excel限界の現場を仕組み化する人** として選んでもらえる。
```

**3つまとめ（15秒）**

```
ターゲットマーケットは、
メッセージを具体化し、時間を集中させ、
「その困りごとなら次廣」と選ばれるために重要だと理解しています。
```

**一覧表（クイック参照）**

| スライドの3メリット | 一言 | 自分（tugilo） |
|---------------------|------|----------------|
| 具体的なメッセージ | 同じ言葉が使える | **4ボヤキ** × WP・121・PT |
| CPに優れた **戦略** | 当たる人に集中 | 経営者・管理者＋士業ルート |
| 競争力の向上 | その課題ならあなた | 構造×時間（AI入口と差別化） |

**BNI・パワーチームで特に効く理由（自分用メモ）**

```
・リファーラルの7割は **コンタクトサークル** —— TM が同じ仲間を見つける軸になる。
・PT は「リストの人」ではなく **同じ TM** のメンバーを選ぶ（梅沢さん＝士業の顧問先、など）。
・TM が曖昧だと、紹介は「とりあえずAI好きな人」になり、双方の時間がもったいない。
・TM が Clear だと、**一人で抱えがちな自分** も、仲間に「こういう人を繋いで」と頼める。
```

**口頭用（30秒・ワークで先に話す）**

```
ターゲットマーケットを決めるのは、営業の勉強というより、
**紹介してくれる人に覚えてもらうため** だと思っています。

私の場合、業種より **症状** ——
「売上伸びなのに Excel 限界」「あの人じゃないと分からない」——
で覚えてもらうと、DragonFly でも PT でも同じ人を指せます。

だから先に「誰の、どんな困りごとか」をはっきりさせてから、
コンタクトサークルやパワーチームのメンバーを選びたいです。
```

#### 2-d What? / Why?（短縮版 — p.5 スライド用）

**What? — 顧客から得られる喜び**

```
お客様から「現場がラクになった」「経営者や管理者が確認に追われなくなった」
「担当が辞めても業務が止まらない」と言っていただけるとき、
「頑張る人が損をしない」状態を一つ増やせたと感じます。
```

**Why? — なぜこのビジネスをしているか**

```
妻と娘2人、猫4匹がいます。仕事が忙しいほど家族時間が削られるので、
「人が頑張らなくても回る仕組み」を作りたいという思いが原点です。
SEとして25年、現場を見ると同じパターンの困りごとが繰り返されていて、
```
