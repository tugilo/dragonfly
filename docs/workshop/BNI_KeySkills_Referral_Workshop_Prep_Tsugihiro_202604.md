## ① 即答カード（最重要・抜き打ち用）

**使い方:** 指名されたら **結論1行だけ** → 止まったら **補足1行**。3秒で思い出すなら **結論の最初の5語** だけでも可。

**Q: リファーラルとリードの違い？**  
→ **結論:** リファーラルは紹介者の信用が乗ったつなぎ。リードは連絡先や噂の切り売りに近い。  
→ **補足:** 誰経由かで、相手の最初の信頼の置き方が変わる。

**Q: レベル1のリファーラルとは？**  
→ **結論:** 名前か噂だけで、困りごとが言えない。  
→ **補足:** 紹介者が第三者に説明できず、次の一歩が決まらない。

**Q: レベル5って何？**  
→ **結論:** いつ・誰と・何を話すかまで決まっていて、紹介された側がその日動ける。  
→ **補足:** 「いつか会おう」止まりはレベル5ではない。

**Q: 今日のゴールは紹介をもらうこと？**  
→ **結論:** いいえ。**紹介されやすい説明と、相手の困りごとの解像度**がゴール。  
→ **補足:** 紹介は結果。操作対象は「言語化」と「型」。

**Q: 商談に繋がらなかったら落ち？**  
→ **結論:** いいえ。**問題が言語化できたか・次の打ち手が一行で言えたか**で勝ち。  
→ **補足:** メンバー面談は商談訪問じゃない。

**Q: 「とりあえず紹介しますね」はアリ？**  
→ **結論:** 危険。**レベルが低いか、紹介者が責任を持ててないサイン**。  
→ **補足:** 紹介者・被紹介者・自分の三本が折れる。

**Q: Givers Gain（ギバーズギャイン）一言で？**  
→ **結論:** **先に与えた人に、質の高い紹介が返る**運用ルール。  
→ **補足:** 信頼残高が、紹介文の解像度になる。

**Q: メンバー1to1で最優先は？**  
→ **結論:** 相手のビジネスを **「誰が・何で・どれくらい困ってる」** と自分の言葉で言い切れること。  
→ **補足:** これがないとレベル3以上の紹介は絶対に作れない。

**Q: 自分の説明で一番やってはいけないこと？**  
→ **結論:** **カテゴリー名だけで終わらせること。**（※名乗りでカテゴリーを言うのは必須。そこから **現場の痛み** まで落とす）  
→ **補足:** 「AI業務改善システム構築の次廣です」で止まると、メンバーは紹介文を書けない。

## ② リファーラルレベル完全整理（瞬時判断用）

| L | 一言定義 | 見分け方（迷ったらここ） | この状態ならレベル〇 |
|---|----------|--------------------------|----------------------|
| **1** | **名だけ・噂だけ** | 「知ってる」「会えるよ」のみ。困りが一言で言えない。 | 第三者に転送しても **何の用か分からない** → **L1** |
| **2** | **連絡先＋ざっくり関心** | 「効率化に興味ある」程度。誰の・どの現場か不明。 | 連絡先はあるが **主語が会社全体のまま** → **L2** |
| **3** | **痛みが具体**（工程・頻度・例がある） | だれが・いつ・月何回、まで言える。 | 「例えば受発注が Excel 交代で、ミスが月○回」まで言える → **L3** |
| **4** | **紹介者が両面にストーリーで説明できる** | 紹介文が **そのまま口頭で10秒** で言える。 | 「御社はまさにこのパターン」と **フィットの一言** がある → **L4** |
| **5** | **次アクションが固定されている** | 日時・参加者・論点のどれかが決まっている。 | 「○日○時、△△さん同席で、××の抜け感を確認」→ **L5** |

**迷ったときの即決ルール**

- **次の一行（打ち手）が書けない** → L2以下。
- **痛みに数字・頻度・担当が付いてない** → L3未満。
- **紹介者が読み上げる文章がない** → L4未満。

**自分について:** こちらの説明が L1〜2 の抽象のままだと、メンバーはどれだけ頑張っても **L3以上を組み立てられない**。だから **現場の言葉まで降ろす**。

## ④ 「紹介される側」完全台本

### 話す順番（型）

**痛み（現場） → 頻度／時間 → キーパーソン → 紹介してほしい相手の絵**

### メンバーがそのままコピペできる紹介ブロック（穴埋め式）

```
【紹介したい経営者・事業主】
・業種／規模：（　　　　　）
・盛り上がっているのは：（売上／案件数／新規 など）
・いま詰まっている現場：（受発注／請求／在庫 など）
・具体的な痛み：（例：夜にExcel転記○時間、ミス月○回、誰か休むと止まる など）
・キーパーソン：（役職＋名前が取れればベスト）

【紹介の一言（そのまま読める）】
「〇〇社の△△さんは、□□が伸びている一方で、××（現場オペ）が Excel と手作業で限界。
　AI業務改善システム構築の次廣さんは、増員せずにそこを仕組みに落とすのが得意なので、一度状況だけ聞いてみてほしい。」
```

### NG例 → 改善例（そのまま練習用）

| NG（紹介者が困る） | 改善（コピペできる粒度） |
|--------------------|---------------------------|
| 「AI やってる人なんで、なんか合うと思います」 | 「**Excel と二重入力で受発注がパンク寸前の成長企業の社長** に会わせたい。**AI業務改善システム構築の次廣さん**はそこを **増員なしで組み替える** のが得意。」 |
| 「システム作れる人です」（抽象） | 「**売上は伸びてるのに事務が追いついてない会社**向け。**業務を分解してから** AI を組み込む。**ツールの使い方講座ではない。**」 |
| 「忙しい社長いませんか？」（主語なし） | 「**受注は月○件まで増えた**が、**発注データの転記が毎晩2時間**、**ミスは月○回**、という社長。」 |
| 「ぜひ営業お願いします」（プレッシャー） | 「**一度10分、現状の流れだけ聞いてもらえれば**」まで落とす。 |

### 自分用・30秒ミニ台本（紹介される側として）

> **AI業務改善システム構築の次廣です。うちは成長企業で、売上は伸びてるんですが、受発注が Excel 交代で、ほぼ毎晩転記が残業で、ミスによる手戻りが月に数回ある状態です。**  
> **経営的には人を増やしたくなくて、誰か一人いなくなると止まる部分をなんとかしたい。**  
> **紹介してほしいのは、同じように「現場オペが売上に追いついてない」経営者です。僕の話を聞いた人が、その人を想像しやすいように具体的に話します。**

## ⑤ 「紹介する側」ヒアリング → 紹介文（一連テンプレ）

### ステップA：質問（この順で飛ばさない）

1. **いま伸びているのは？**（売上・件数・新規など）  
2. **その伸びをさばく側で、いちばんパンクしてるのはどの工程？**（部署名まで）  
3. **そこで、月何回／何人・何時間、手戻りやミスが出てる？**（わからなければ「多い／毎回」でも可）  
4. **Excel・手メモ・チャット・別ツールを「またいで」二重入力してる箇所は？**  
5. **明日その担当が一人辞めたら、真っ先に止まるのは？**  
6. **社長は来年、増員と仕組み、どっちに寄せたい？ 阻んでる現実は？**

### ステップB：整理シート（メモ用・30秒で埋める）

```
誰が：〔　役職／人物　〕
どの業務で：〔　受発注／請求／在庫 など　〕
何が起きてる：〔　転記／二重入力／ミス／残業　〕＋〔　頻度：月○回・毎晩○時間 など　〕
経営の意志：〔　増員したくない／人が足りない　〕
```

### ステップC：紹介文生成（レベル4想定・型）

**型:**

> **〔会社／役職〕の〔名前〕さんは、〔伸びている指標〕がある一方で、〔工程〕が〔痛みの具体〕で限界。**  
> **AI業務改善システム構築の次廣さんは、増員せずにそこを仕組みに落とすのが得意で、Excel と二重入力でパンクしがちな成長企業の経営者に強いので、一度状況だけ聞いてみてほしい。**

**実例（架空で型確認）:**

> **山田製作の営業部長の山田さんは、受注が前年比130%な一方で、受発注が Excel 三本立ての転記で、ミスによる出荷訂正が月に5回出ている。**  
> **AI業務改善システム構築の次廣さんは増員前にオペを組み替えるのが得意なので、一度10分だけ流れを聞いてみてほしい。**

### 狙うレベル

- **最低 L3**（痛みが具体）まで必ず上げる。  
- 余裕があれば **L4**（上の紹介文まで一緒に作る）。**L5 は無理に狙わない**（相手の都合が要る）。

## ⑦ ワークショップ攻略メモ（本番）

### 評価される人の特徴

- 指名されたら **0.5秒で結論**。
- 相手の話を **「誰が・何で・どれくらい」** に即変換できる。
- **紹介文を口頭で披露できる**（自分分・相手分どちらでも）。
- **個人事業主**も紹介候補に含め、**困りごと＋効果**を一言足せる（§⑧）。

### その場で差がつく動き

- **ホワイトボード／メモを見せながら** 構造化する（ファシリ感）。
- 相手のビジネスに **深い質問を1つ入れる**（浅い世間話は切る）。
- 「**今日の自分の成果は、紹介文が一行できたか**」で振り返る。

### やってはいけないこと

- 長い自慢・実績羅列。
- **「紹介ちょうだい」** の圧。
- 相手の話を遮って **自分の技術話**にすり替える。
- レベルを上げずに **とりあえず名刺** だけ増やす動き。

### 一段上に見せるコツ

- **「結論、言います」** と宣言してから答える。
- 質問に答えたあと **「なのでレベル〇です」** と番号で締める（教官感は出しすぎない）。
- **相手の紹介文を代人作文して差し出す**（Givers Gain の見せ場）。

## ⑧ 事例と効果（紹介を一段上げる・WSフィードバック）

**狙い:** メンバーが第三者に渡すとき、**役割名だけ**だと想像しづらい。**短い場面＋そのあとの変化**まで入れるとレベルが上がる。

**ルール:** 実在の固有名は出さない。**困り → やったことのイメージ → 変化** の順で1〜2文。詳細な型・全文は [レッスン2 Script「■事例と効果」](./BNI_KeySkills_Lesson2_Referral_Script_Tsugihiro_202604.md) を正本とする。

**口頭用ミニ例（匿名）**

1. **個人事業主** — 請求と入金を紙と表計算の両方でつけていて夜までかかる → 書く場所を一つにまとめた → **月末の確認が短くなり、抜けが減った**。  
2. **小さな会社** — 注文内容をチャットと表計算の二重で管理 → 流れを一本化した → **担当が休んでも続けられる**。  
3. **よくある現場** — 毎晩の写し直しが残業の原因 → 手順を分けてつなぎ目だけ短くした → **残業が目に見えて減った**。

**効果を短く言う語彙:** 時間に余裕・手戻り減・同じ作業を二回やらない・一人社長の夜が楽、など。
