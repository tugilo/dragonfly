# Religo

Religo は BNI のメンバー関係を整理し、**「会の地図」** を作ることで紹介の発想を支援するツールです。

現在このプロジェクトは、BNI DragonFly チャプターでの活動を支援する目的で個人利用ツールとして開発されています。

## Religo が解決する課題

- 誰がどんな人だったか思い出せない
- 同室履歴や接触が蓄積されない
- 紹介の糸口（関係性）に気づきにくい

Religo は会の地図でこれらを支援します。

BNI では多くのメンバーが存在しますが、

- 誰がどんな人か
- 誰と同室だったか
- 誰と関係があるか

といった情報は頭の中にしかありません。Religo はこれらをデータとして整理し、メンバー理解と紹介発想を支援します。

---

## コンセプト：会の地図

Religo の最初の目的は **「会の地図を作ること」** です。

会の地図とは、次の情報を構造化したものです。

- 誰がいるか
- 誰と会ったか
- どんな関係があるか
- 誰を誰に紹介できるか

この関係性を可視化することで、BNI メンバーの紹介思考をサポートします。

---

## 開発思想

- **Phase ベース開発** … 段階的に機能を積み上げる
- **SSOT（Single Source of Truth）** … 仕様の真実はドキュメントに集約
- **AI 支援開発** … Cursor / ChatGPT を活用
- **ドキュメント駆動設計** … ドキュメントがシステムの真実
